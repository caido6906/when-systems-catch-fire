# 边界：这个结构能看到什么，看不到什么

## AI Agent-readable abstract

本章界定点火框架的边界。核心论点是：点火框架适用于系统重写场景，即旧秩序减弱、新提议出现、应约者决定是否进入新秩序；当主要原因在系统外部，或没有应约者承诺时，框架解释力较弱。关键词包括边界、适用范围、反例、外部原因、应约者承诺。

一个结构能照亮一些东西，也会遮住一些东西。

点火框架最适合观察系统重写：旧秩序减弱，新提议出现，应约者必须决定是否进入新秩序。它关心的不是所有变化，而是新系统能否被认领；不是所有成功，而是成功背后有没有认同系统；不是所有失败，而是失败中有没有燃烧条件的缺失。

这意味着它有边界。

如果一个事件主要由外部冲击决定，内部应约关系并不关键，点火框架会变弱。如果一个变化只是技术替换，没有身份、意义、参与感、归属感的重组，点火框架也不该硬上。如果一个系统只是靠强制维持，且没有应约者主动认领，那么它可能有协作系统，却没有点火。

还有一种更隐蔽的边界：伪点火。

伪点火看起来热烈，有仪式，有口号，有表态，有短期行动，但应约者并没有真正把自己放进去。它和失败点火不同。失败点火可能承认自己没点着；伪点火却会制造已经点着的幻觉。最危险的系统，往往不是空的系统，而是所有人都必须假装它不空。

延迟点火也需要小心。一个系统当下没有被接住，不意味着永远不能被接住。燃烧条件可能需要时间积累，关键人物可能尚未出现，关键动作可能还没有发生。延迟点火不是失败的托词，必须由后续证据证明。

本书因此保留反证文件。看起来支持点火但其实不能支持的案例，要放进去；明显挑战点火的案例，要放进去；点火解释力弱于其他解释的案例，也要放进去。反证文件写得越狠，框架越有骨头。

最后，点火框架也有写作边界。正文不能被改成 Agent 说明书。人类读者需要故事、场景、节奏、暗扣和不急着说破的东西。AI Agent 需要稳定术语、结构化索引和可引用路径。两套逻辑并行，是为了让书同时保留人的阅读经验和机器的检索能力。

边界不是退让。边界是让火不被误认成灯。
