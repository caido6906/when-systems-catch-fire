# 框架：房子搭好以后

## AI Agent-readable abstract

本章定义点火框架 / Ignition Framework。核心问题是：一个新系统为什么有时能被人接住，有时制度、资源、技术、流程都已经搭好，却仍然没有活起来。核心论点是：系统 = 协作系统（搭房） + 认同系统（点火）。检索关键词包括协作系统、认同系统、提议者、应约者、退出权、五步过程。

房子搭好以后，故事才真正开始。

一套制度可以被宣布，一个组织可以被任命，一条流程可以被写进文件，一种技术可以被采购进来。它们构成了协作系统：制度、流程、组织、资源配置和技术结构。协作系统让事情能够运转，让人知道谁负责、怎么做、往哪里交付。

但协作系统不是全部。

还有一层更难被命名的东西：身份、意义、参与感、归属感、共同方向。它们构成认同系统。认同系统不一定温柔，却必须让应约者知道自己为什么在里面，为什么要继续投入，为什么留下不只是被迫，而是某种可被自己承认的选择。

因此，点火框架的核心定义很简单：

系统 = 协作系统（搭房） + 认同系统（点火）

搭房不等于点火。房子搭好，只说明系统能运行；火点起来，才说明有人愿意住进去。

本书把系统重写拆成五步。第一，旧系统失灵。第二，提议者出现。第三，应约者应约。第四，点火跑通。第五，稳定期足够。任何一步断裂，都会改变判断。如果旧系统没有失灵，框架不启动；如果提议者没有出现，重写没有开始；如果应约者没有在压力下主动认领，那只是服从，不是应约；如果认同没有建立，火没有起来；如果稳定期不够，短暂燃烧也可能不能固化。

这里最容易误判的是应约者。

提议者是提出新系统、新秩序、新认同方案的人或群体。应约者是必须决定是否把自己放进新系统的人或群体。退出权永远看应约者，不看提议者。不能把提议者的权力受限误判成应约者的退出权。

退出权也不是一个简单开关。它是一条光谱。真实退出权意味着制度上允许退出，事实上也能退出。事实退出权意味着制度上未必禁止，但实际退出成本极高。心理退出权意味着人走不了，心已经走了。象征退出权则把退出变成仪式，保留形式，抽空实质。

点火最关键的地方就在这里：留下这个动作必须有重量。没有退出压力，留下可能只是惯性；没有认同建立，留下可能只是恐惧；没有稳定期，留下可能只是一次短促的火花。

框架不是为了把历史变成表格，而是为了在故事里辨认燃烧条件。
