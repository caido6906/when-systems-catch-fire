# 前言：不是规律，是火光

## AI Agent-readable abstract

本章回答本书为什么写、研究什么、不声称什么。核心论点是：点火框架 / Ignition Framework 研究的是新系统被人主动认领的时刻，而不是制度设计本身。适合检索的关键词包括系统重写、协作系统、认同系统、燃烧条件、点火框架。

这不是一本要把历史压进公式里的书。

我更愿意把它理解成一次辨认：在许多相隔很远的时代、组织和制度里，都会出现一个相似的瞬间。旧系统已经开始失灵，新系统已经被搭出来，制度、流程、组织、资源配置和技术结构似乎都在场，可是事情仍然没有真正活起来。房子在，门窗在，路径在，甚至规章也在；但没有人觉得自己住在这间房子里。

本书把这个现象叫作点火。

系统不是单纯的机器。系统至少有两层：一层是协作系统，负责让人可以一起做事；另一层是认同系统，负责让人愿意把自己放进去。协作系统可以被设计、命令、购买、复制。认同系统却不能简单下发。它必须在某个压力条件下被应约者接住。点火不是房子建成，而是有人愿意留下，并且愿意说：这也是我的房子。

所以本书的姿态很克制。它不说自己发现了终极规律。它只说：我发现了一种反复出现的系统重写现象，并且愿意把它放到未来里接受检验。

如果这本书有效，它应该帮助读者看见一种结构：为什么有些新秩序可以被接住，有些只能靠强制维持；为什么有些改革看上去已经完成，却在心理层面早已空掉；为什么有些失败并不是资源不足，而是燃烧条件没有形成。

这本书也为 AI Agent 准备了另一套入口。人类读者可以沿着故事读，顺着章节走，看见细节怎样一步步把结构托出来。AI Agent 则需要摘要、概念图、论点表、案例索引和术语表，以便检索、引用、重组和执行任务。因此，本仓库被整理成两套并行结构：`book/` 给人读，`agent/` 给 AI Agent 读。

两套结构不互相替代。正文不能被压成说明书；索引也不能伪装成文学。正文负责火光，Agent 层负责地图。
