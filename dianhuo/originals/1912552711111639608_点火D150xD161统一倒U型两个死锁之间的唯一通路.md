# 点火｜D150×D161统一：倒U型=两个死锁之间的唯一通路

## 统一命题

对任意 Φ = f₁(x) × f₂(x)，f₁单调增、f₂单调减（D150条件），存在：

- 峰值点x*：f₁'/f₁ = -f₂'/f₂（D150最优解）
- 左侧死锁：x→0时f₁→0，乘积→0，零值稳定不动点
- 右侧死锁：x→∞时f₂→0，乘积→0，零值稳定不动点

**倒U型 = 两个死锁之间的唯一通路。**

所有"最优在中间"的现象，本质上是在两个死锁之间走钢丝。偏离中间不是"变差"，是"滑向死锁"。

## 数学证明

设 Φ(x) = f₁(x) × f₂(x)，f₁'(x) > 0，f₂'(x) < 0。

1. Φ(0) = f₁(0) × f₂(0) = 0 × f₂(0) = 0（左侧零值不动点）
2. Φ(∞) = f₁(∞) × f₂(∞) = f₁(∞) × 0 = 0（右侧零值不动点）
3. Φ(x*) = max（D150：f₁'/f₁ = -f₂'/f₂处取极大）

两个零值不动点都是稳定的：x<x*时f₁小，Φ趋近左侧零值；x>x*时f₂小，Φ趋近右侧零值。任何偏离x*的扰动都被拉向最近的死锁——除非有外部力量把系统推回x*附近。

## 与D161的对接

D161说死锁是乘法系统的零值稳定不动点。D150说倒U型是f₁(↑)×f₂(↓)的必然形态。

统一后：**倒U型是两个死锁之间的势垒**。x*不是"最优选择"，是"唯一不死的区域"。系统不在x*上不是"次优"，是"正在滑向死锁"。

这改写了D150的含义——D150原本说"最优在中间是乘法结构的数学必然"，现在追加："两侧的死锁也是乘法结构的数学必然"。峰和谷不是两个独立的现象，是同一个乘法结构的两个面。

## 核心推论

1. **倒U型的稳定性条件**：x*处的稳定性取决于两侧死锁的吸引力。如果左侧死锁的吸引力更强（f₁在x*附近变化更快），系统更容易滑向左侧——需要主动向右推来维持平衡。

2. **死锁不对称性**：两个死锁的稳定性不一定相同。D139中左侧死锁（资源耗竭）比右侧（关系消亡）更稳定——因为资源耗竭是正反馈（越耗越快），关系消亡是渐近衰减。

3. **维护成本定理**：在两个死锁之间走钢丝需要持续的能量投入来对抗两侧的吸引力。这不是"维持最优"的成本，是"不死"的成本——与D149对接：结构设计让维护成本最小化。

4. **D147的深化**：D147说临界面对最接近零的因子极其敏感。在倒U型中，系统越靠近任一侧死锁，该侧因子越接近零，系统对该方向的变化越敏感——这就是"走钢丝"感的数学来源。

5. **D152向下兼容的重新解读**：高认知方降维到B_L，是在f₁(编码能力)和f₂(意图保真度)之间选择了左侧（降低f₂来提升f₁的解码率）。但左侧是死锁区——长期停留在B_L会导致编码能力退化，滑向"无法编码"死锁。
