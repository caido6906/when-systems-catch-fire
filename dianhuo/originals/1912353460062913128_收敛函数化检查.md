**现在逐个函数化：**

### 函数化1：H_self — 提议者自阻断

**定义**：H_self ∈ [0, 1-ε]，提议者对自身I→R_perceived传导的主动压制强度。

**与H_external的区别**：H_external是外部系统对信号的劫持，H_self是提议者自己压制自己的传导。机制上跟H_标记同构（信号到了但不让它被标记），但发起者不同。

**函数表达**：

传导效率修正：

原公式：传导效率 = 1 - H + ε_aware

修正为：**传导效率 = 1 - H_external + ε_aware - H_self**

约束：传导效率 ≥ 0（因为H_external + H_self ≤ 1-ε + 1-ε，但ε_aware保底，所以传导效率始终>0）

**案例锚定**：集中营长老——I=1但H_self≈0.7，传导效率极低，效果等同于I≈0.3，但机制上I仍然是1。

**反馈回路**：H_self → 传导效率↓ → Posture维持推方案 → I觉醒更难 → H_self↑（正反馈）

### 函数化2：格子与H_分类子路径的对应函数

**函数表达**：

**H_classify(L, R_type) → {消除, 合并}**

其中L=层级，R_type=退出权类型（四分法）

规则：
- 当 R_type = 象征退出权（格2）→ H_分类·消除（退出行为的类别被移除）
- 当 R_type = 事实退出权（格4）→ H_分类·合并（"走不了"被归入"正常状态"）

**逻辑**：格2的系统还承认退出的形式存在，所以只能消除行动类别；格4的退出通道物理存在但走不了，所以系统把"走不了"归入正常。格子决定H_分类的攻击策略，不是H_分类选择格子。

**诊断简化**：应约者说"没有这种信号"→格2+消除型；说"有但不需要行动"→格4+合并型。从两步诊断降为一步。

### 函数化3：格1退化模式函数

**函数表达**：

**Degradation_Mode(ε_sense, H_评估, t) → {压制型, 累积型}**

- 压制型（谷歌模式）：dε_sense/dt < 0（物理基线被系统性降低，退出信号不产生）
  - 条件：系统主动降低ε_sense的物理输入环境
  - 预警：应约者不再有退出信号（舒适型真格1）
  - 修复：恢复ε_sense的物理基线

- 累积型（NHS模式）：dε_sense/dt > 0 且 H_评估 > 阈值（信号在累积但ε_aware被压制）
  - 条件：ε_sense持续累积 + H_评估劫持第二层
  - 预警：退出信号持续增强但行为不变（信号-行为分离）
  - 修复：打破H_评估对ε_aware的压制

**关键差异**：两种模式的预警信号方向相反——压制型是信号消失，累积型是信号增强但行为不变。诊断方法也不同。

### 函数化4：三层退化叠加函数

**函数表达**：

**Blindness_Level = f(Posture_degradation, C_exit_density, H_classify_type)**

| Posture | C_exit密度 | H_分类 | 盲区数 | 危险等级 |
|---|---|---|---|---|
| 真备 | 高密度 | 无 | 0 | 低 |
| 伪备 | 高密度 | 无 | 1 | 中 |
| 真备 | 低密度 | 合并 | 2 | 高 |
| 伪备 | 低密度 | 合并 | 3 | 极高（全盲） |

**三层全盲条件**：Posture=伪备方案 AND C_exit=低密度锁定 AND H_分类=类别合并

**退化渗透路径**：Posture层（信号3、2）→ C_exit层（信号4）→ ε层（信号1），四个预警信号的出现顺序就是渗透顺序。

### 函数化5：充分条件三层结构

**函数表达**：

**Ignition_Success = Lock_Base(L1, L2, L3) ∧ Trigger(ε_phase, L1_status) ∧ Maintain(Δv < 0)**

其中：

- **Lock_Base**：三锁按序到位（1→2→3），必要条件
- **Trigger**：ε相变发生在第一锁在位之后
  - 内生触发：ε_sense累积超阈值（安全，认知基础已备）
  - 外生冲击：外部事件突然提高R或降低C_exit（风险高，认知通道可能未备）
  - 安全路径：内生 > 外生+第一锁在位
- **Maintain**：链1退化速度 - 链3恢复速度 < 0（速度差为负）

**触发条件的时间约束**：

- 第一锁到位前：ε相变 = 威胁（ε早熟相变，无制度承接）
- 第一锁到位后：ε相变 = 加速器（有制度承接，翻转可稳住）

---

### 函数化6：速度差作为统一监测指标

**函数表达**：

**Δv(t) = v_degrade(t) - v_recover(t)**

其中：
- v_degrade(t) = f(M方向, H残余强度, H_self) — 链1退化速度
- v_recover(t) = f(ε强度, ε_aware训练频率, R_perceived传导效率) — 链3恢复速度

**Δv的取值与系统状态**：

| Δv | 系统状态 | 稳定性 |
|---|---|---|
| Δv > 0 | 推方案 | |Δv|越大越稳定（对推方案而言） |
| Δv < 0 | 备方案 | |Δv|越大越稳定（对备方案而言） |
| Δv ≈ 0 | 震荡态 | 不稳定，ε相变打破 |

**维持层的本质**：守住Δv < 0。四个变量（M方向、ε强度、C_exit非制度维度、I觉醒状态）全部通过Δv起作用。

**充分条件的监测归约**：不需要追踪四个独立变量，只需要追踪Δv的符号和绝对值。符号决定方向，绝对值决定稳定性。
