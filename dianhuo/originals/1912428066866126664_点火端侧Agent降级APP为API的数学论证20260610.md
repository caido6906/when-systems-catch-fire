# 点火｜端侧Agent降级APP为API的数学论证（2026.06.10）

## 问题
AIPC已出现，端侧Agent正在铺开。所有智能设备最终都会自带Agent运行。不Agent化的APP会被端侧Agent降级为API调用——这是第三条致死路径。数学化推导。

## 推导

### 第一步：端侧Agent改变用户-APP关系模型

传统：用户 → APP界面 → 服务。P_infra(APP) = min(1, P_interface, P_action)
端侧Agent化后：用户 → 端侧Agent → APP的API → 服务。P_infra(APP) = min(1, P_API, P_agent_action)

关键跳变：P_interface被P_API替代。界面交互价值归零。端侧Agent绕过界面，用户不再直接操作APP。

### 第二步：APP价值函数被端侧Agent重写

传统价值：V_APP = Q_content × P_interface × P_action
端侧Agent化后：
- 不Agent化的APP：P_interface→0（端侧Agent绕过界面），P_API=1（有数据接口即可调用），P_agent_action=1（端侧Agent自己能做事）。V_APP(不Agent化) = Q_content × 1 × 1 = Q_content。APP退化成纯数据源，价值只剩内容质量。
- Agent化的APP：额外拥有P_agent_own=1（自有Agent能力，能主动服务、学习、预判）。V_APP(Agent化) = Q_content × 2。

### 第三步：主体性函数——从主体到客体

S(t) = P_initiative(t) × P_control(t) × P_data_flow(t)

- P_initiative：能否主动发起服务（推送、预判、提醒）
- P_control：能否控制用户交互路径
- P_data_flow：数据流向是双向还是单向

传统APP：S = 1 × 1 × 1 = 1
被API降级后：S = 0 × 0 × 0.5 = 0（不能主动发起、不能控制路径、数据只能被动被拉取）

端侧Agent接管后，APP的主体性从1归零。不再是服务提供者，而是数据供应商。

### 第四步：端侧Agent扩散时间动力学

AIPC/端侧Agent渗透率N_device(t)服从logistic增长：
N_device(t) = N_max / (1 + exp(-k_device × (t - t_mid)))

API降级概率：
P_API_demotion(t) = N_device(t) / N_max = 1 / (1 + exp(-k_device × (t - t_mid)))

N_device/N_max → 1时，P_API_demotion → 1，所有APP都被降级为API调用。

### 第五步：与P_forward统一函数对接

端侧Agent化对四因子的冲击：

1. σ(ε×A×D×|M_cog|)：端侧Agent替用户完成认知跳变，APP不再参与认知过程→贡献归零
2. exp(-n_lock×C̄/θ_C)：界面锁定消失（用户不操作界面了），n_lock至少减1；数据锁定被端侧Agent缓存/索引能力削弱
3. f(P_infra)：APP的P_infra被端侧Agent接管，自身P_infra不再有意义
4. σ(θ_d - d(t))：端侧Agent可无缝切换数据源，d(t)加速趋零

综合：P_forward(APP, t) = P_forward_0 × (1 - P_API_demotion(t)) × σ_decay(n_lock - Δn_lock)

Δn_lock是端侧Agent消除的锁定维度数（≥1）。t→∞时P_API_demotion→1→P_forward→0。

### 第六步：双路径乘法叠加

路径1（已有结论）：竞品Agent化拉低d(t)，σ(θ_d - d_relative(t))→0
路径2（本轮新增）：端侧Agent降级APP为API，(1 - P_API_demotion(t))→0

P_survival(APP, t) = P_forward_0 × σ(θ_d - d_relative(t)) × (1 - 1/(1 + exp(-k_device × (t - t_mid))))

两条路径独立作用、乘法叠加。任一归零则P_survival归零。t→∞时双路径同时趋零，P_survival→0是确定性事件。

## 收敛结论

不Agent化的APP被两条独立的数学力量同时绞杀：竞品在拉低退出代价d(t)，端侧Agent在剥夺主体性S(t)。不是"慢慢失去竞争力"，而是被确定性因果链锁死——P_survival单调递减，穿过零阈值只是时间问题。

端侧Agent的扩散速率k_device由硬件替换周期决定（AIPC渗透→手机端Agent渗透→IoT端Agent渗透），是物理约束而非商业选择，不可被任何APP的策略逆转。

APP Agent化的窗口期 = 端侧Agent从0到50%渗透的时间区间。过了这个窗口，P_API_demotion过半，APP的主体性已被端侧Agent接管大半，再Agent化也只是"端侧Agent的一个可选后端"而非"用户的智能伙伴"。
