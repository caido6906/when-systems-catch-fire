# 点火框架三层结构：条件·方向·状态

点火框架由三层独立但因果关联的结构组成。24格矩阵管条件，6种方向管认同锚点，5种状态管火种存续。三层各管各的，合在一起覆盖全部67个案例，没有遗漏也没有冗余。

## 第一层：条件（24格矩阵）

点火那一刻的系统配置。三个输入变量 × 一个结果变量：

- 提议者意识：有意识 / 无意识
- 提议者退出权：有 / 无
- 应约者退出权：有 / 无
- 应约者状态：积极 / 消极 / 无人（结果变量，由前三个和方案质量共同决定）

3 × 2 × 2 × 2 = 24格。实际有案例的格子约15个，其余是"理论上可能但历史上没出现"的配置。

24格矩阵只管条件——它告诉你点火那一刻系统长什么样，不告诉你火点着了没有、方向对不对、火还活着没有。

## 第二层：方向（6种）

认同建在哪。独立于火的状态——火可以点着了但方向歪了，也可以方向对了但火灭了。

| 方向 | 含义 | 稳定性 | 案例 |
|---|---|---|---|
| 建在应约者身上 | 应约者自己会维持 | 最稳 | 周公、穆罕默德、华为、美国建国 |
| 建在部分应约者身上 | 没覆盖的那部分随时可能退出 | 维持断裂 | 彼得大帝（只点到贵族）、凯末尔（只点到城市精英） |
| 建在外部 | 外部条件一变就散 | 脆 | 唐朝（建在开放本身上）、明治维新（建在脱亚入欧上） |
| 建在个人权威上 | 人一走认同就散 | 危 | 庇隆主义、海尔·塞拉西 |
| 建在旧系统的否定上 | 旧认同拆了新的还没建 | 烧自己房子 | 戈尔巴乔夫、法国大革命初期 |
| 没建 | 没点火，无方向 | 散 | 秦、元朝、蒙古帝国 |

方向与之前"五种点火趋势"的关系：
- 点对了 = 建在应约者身上
- 点不足 = 建在部分应约者身上
- 点歪了 = 建在外部 或 建在个人权威上
- 点反了 = 建在旧系统的否定上
- 没点火 = 没建

方向是结果判定，不是事前标签。不能在点火的那一刻就判方向，要等维持力量跑一段才能看清认同建在了哪里。

## 第三层：状态（5种）

火点着了没有、还活着没有。独立于方向——方向对了但火可能灭了，方向歪了但火可能还活着。

| 状态 | 含义 | 维持力量 | 案例 |
|---|---|---|---|
| 点着了·活着 | 点火成功，火种在跑 | 硬件+软件至少一种在 | 周公、穆罕默德、美国建国、华为、清朝 |
| 点着了·灭了 | 点火成功，但维持力量耗尽 | 两种都没了或被外力中断 | 阿育王（死后迅速衰退）、乌鲁卡吉纳（被外力碾碎） |
| 没点着·火种还在 | 点火失败，但硬件或软件还在跑，可以被后人重新点 | 至少一种在 | 孔子（300年后汉武帝补火）、秦（汉补火）、元朝（朱元璋接上） |
| 没点着·火种也没了 | 点火失败，维持力量也没了 | 两种都没了 | 戈尔巴乔夫（旧认同拆了新的没建）、卡拉卡拉（旧激励灭了新的没点着） |
| 从未点火 | 提议者缺席，连点火动作都没有 | 无 | 蒙古帝国、印度种姓制度、非洲班图 |

维持力量的判断标准：
- 硬件：搭房留下的制度、基础设施、行政框架
- 软件：点火留下的认同、文化、价值观
- 两种都在 → 火没灭（元朝97年华夏没断，硬件软件都在跑）
- 只有一种 → 火在摇（秦汉之间靠硬件，孔汉之间靠软件）
- 两种都没了 → 火灭了（西罗马亡后公民权断了认同也断了）

## 三层之间的关系

条件 → 方向 → 状态，有因果关系但不是一一对应：

- 同样的条件可能导向不同的方向和状态（格10可以点对也可以点歪）
- 同样的方向在不同条件下状态不同（建在应约者身上+格1=点着了活着，建在应约者身上+格4=点着了但温室里没验证过）
- 格子漂移让同一个案例在不同时间点处于不同的条件、方向、状态

三层结构不替代24格矩阵和五种点火趋势，而是把它们重新组织为更清晰的逻辑：条件管输入，方向管锚点，状态管存续。

## 矩阵精度判定

24格矩阵精度足够，不增不减：
- 三个维度（提议者意识、提议者退出权、应约者退出权）是点火条件里真正影响结果的变量
- 67个案例全部能落格，没有放不进去的
- 格子漂移解释了时间维度上的变化，不需要再加时间轴
- 空格不是问题——它们是"理论上可能但历史上没出现"的配置
- 方向和状态从24格矩阵中独立出来后，24格只管条件，职责更清晰
