# 《点火》框架·GPT校准补丁与全量重标总表·2026.06.03 21:43

## 一、术语修正

秦代"象征退出权=强制力的优雅版本"改为"象征退出权=强制力的仪式化版本"。秦是赤裸裸暴力，不伪装，"优雅"不准确。

## 二、表述校准

“五项史实全部支撑框架，未发现反例”→ 改为"五项核查没有推翻框架，反而迫使框架完成了更精细的校准"。强调框架的容错升级能力，不做过满判断。

## 三、应约者跨尺度拆分

应约者不是同一种人，需按尺度区分：

-   文明级：多个国家、族群、阶层

-   国家级：公民、官僚、地方势力

-   政策级：一线执行者、被治理者

-   公司级：员工、用户、投资人

不拆的话，读者会觉得你在不同层级之间滑动。

## 四、反证文件 [counter-cases.md](http://counter-cases.md)

GPT建议专门建一份反证文件，只放三类：

1.  看起来支持点火，但其实不能支持的案例

2.  明显挑战点火的案例

3.  点火解释力弱于其他解释的案例（财政、军事、地理、技术、人口结构更关键的）

最大风险不是没有材料，而是材料太多，每个案例都被吸进同一个结构里。反证文件写得足够狠，框架才有骨头。

## 五、GitHub仓库三层目录

```auto
01-manuscript/     # 给读者看的书稿
02-framework/      # 框架定义、变量、矩阵、退出权四分法
03-evidence/       # 案例库、反例、预测、史实核查
  ├── fact-checks/
  ├── calibration-log/
  ├── predictions/  # 每个预测单独存档，只新增复盘不覆盖原文
  └── reviews/

```

03-evidence/ 最关键——书能不能站住，不只靠正文好看，靠背后有一套能追溯的校准系统。

## 六、预测文件固定格式

每个预测文件四块：

1.  案例名称

2.  当前状态

3.  预测条件句

4.  我判断错了的标准

预测必须留原始版本，不能事后改。后面只能新增复盘文件，不能覆盖原文。
