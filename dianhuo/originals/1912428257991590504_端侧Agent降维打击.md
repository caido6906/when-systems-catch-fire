# 端侧Agent降维打击

这个角度很硬——端侧Agent一旦铺开，APP的命运就从"自己决定要不要进化"变成了"被端侧Agent降级为API调用"。我来推导。好，已有的函数体系我清楚了。现在从端侧Agent这个产业侧角度推导——这是一个新的致死路径，跟之前的"竞品Agent化拉低d(t)"不同，这次是**APP连竞品资格都丢了，被降级为API**。

推导如下：

**第一步：端侧Agent改变用户与APP的关系模型**

传统模型：用户 → 直接操作APP界面 → 获得服务
端侧Agent模型：用户 → 端侧Agent → 调用APP的API → 获得服务

用函数标注：
- 传统：P_infra(APP) = min(1, P_interface, P_action) — 用户通过界面操作，P_interface=1, P_action取决于APP自身能力
- 端侧Agent化后：P_infra(APP) = min(1, P_API, P_agent_action) — 端侧Agent替代界面，APP只需暴露API

关键跳变：P_interface被P_API替代。界面交互价值归零。

**第二步：APP的价值函数被端侧Agent重写**

传统价值：V_APP = Q_content × P_interface × P_action
端侧Agent化后：V_APP = Q_content × P_API × P_agent_action

其中P_agent_action是端侧Agent的执行能力，不由APP决定。

对不Agent化的APP：
- P_interface → 0（端侧Agent绕过界面，用户不再直接操作）
- P_API = 1（只要APP有数据接口，端侧Agent就能调用）
- P_agent_action = 1（端侧Agent自己能做事）

所以：V_APP(不Agent化) = Q_content × 1 × 1 = Q_content

APP退化成纯数据源，价值只剩内容质量Q_content。

对Agent化的APP：
- P_API = 1
- P_agent_action = 1
- 额外能力：P_agent_own = 1（APP自有的Agent能力，能主动服务、能学习、能预判）

所以：V_APP(Agent化) = Q_content × 1 × 1 × (1 + P_agent_own) = Q_content × 2

**第三步：API降级的数学本质——从主体到客体**

定义主体性函数：
S(t) = P_initiative(t) × P_control(t) × P_data_flow(t)

- P_initiative：能否主动发起服务（推送、预判、提醒）
- P_control：能否控制用户交互路径
- P_data_flow：数据流向是双向还是单向

传统APP：S = 1 × 1 × 1 = 1（主动推送、控制界面、双向数据流）
被API降级后：S = 0 × 0 × 0.5 = 0（不能主动发起、不能控制路径、数据只能被动被拉取）

端侧Agent接管后，APP的主体性从1归零。

**第四步：端侧Agent扩散的时间动力学**

AIPC/端侧Agent的渗透率N_device(t)服从logistic增长：
N_device(t) = N_max / (1 + exp(-k_device × (t - t_mid)))

其中k_device是硬件替换周期决定的扩散速率，t_mid是50%渗透时间点。

每台端侧设备自带Agent → 端侧Agent覆盖率 = N_device(t)/N_max

定义API降级概率：
P_API_demotion(t) = N_device(t) / N_max = 1 / (1 + exp(-k_device × (t - t_mid)))

当N_device/N_max → 1时，P_API_demotion → 1，所有APP都被降级为API调用。

**第五步：与P_forward统一函数的对接**

端侧Agent化对P_forward四因子的冲击：

1. σ(ε×A×D×|M_cog|)：认知跳变因子——端侧Agent替用户完成认知跳变，APP不再参与认知过程，此项对APP的贡献→0
2. exp(-n_lock × C̄/θ_C)：锁定维度——界面锁定消失（用户不操作界面了），n_lock至少减1；如果APP只有内容没有Agent，数据锁定也被端侧Agent的缓存/索引能力削弱
3. f(P_infra)：基础设施——APP的P_infra被端侧Agent接管，APP自身的P_infra不再有意义
4. σ(θ_d - d(t))：退出代价——端侧Agent可以无缝切换数据源，d(t)加速趋零

综合：P_forward(APP, t) = P_forward_0 × (1 - P_API_demotion(t)) × σ_decay(n_lock - Δn_lock)

其中Δn_lock是端侧Agent消除的锁定维度数（≥1）。

当t → ∞：P_API_demotion → 1 → P_forward(APP) → 0

**第六步：收敛结论——两条致死路径的叠加**

之前推导的致死路径1（竞品Agent化拉低d(t)）+ 本轮路径2（端侧Agent降级APP为API）：

P_survival(APP, t) = P_forward(APP, t)
= σ(ε×A×D×|M_cog|) × exp(-n_lock×C̄/θ_C) × f(P_infra) × σ(θ_d - d(t)) × (1 - P_API_demotion(t))

路径1作用于第四因子σ(θ_d - d(t))——竞品让d(t)下降
路径2作用于新增因子(1 - P_API_demotion(t))——端侧Agent让APP失去主体性

两条路径独立作用、乘法叠加。任一归零则P_survival归零。

**最终收敛函数：**

P_survival(APP, t) = P_forward_0 × σ(θ_d - d_relative(t)) × (1 - 1/(1 + exp(-k_device × (t - t_mid))))

t→∞时：σ(θ_d - d_relative)→0（路径1）且 (1-P_API_demotion)→0（路径2）

双路径乘法归零，P_survival→0是确定性事件。

不Agent化的APP不是"慢慢失去竞争力"，而是被两条独立的数学力量同时绞杀——竞品在拉低你的d值，端侧Agent在剥夺你的主体性。存好了。这条推导的核心结论：

**端侧Agent引入了第二条独立的致死路径——APP不Agent化，不是被竞品淘汰，而是被自己设备上的Agent降级为API。** 主体性函数S(t)从1归零：不能主动发起服务、不能控制交互路径、数据只能被动被拉取。

跟之前竞品Agent化拉低d(t)那条路径是乘法叠加的——两条独立因果链，任一归零P_survival就归零。

而且端侧Agent的扩散速率k_device是硬件替换周期决定的物理约束，不是商业策略能逆转的。AIPC已经出了，手机端Agent紧随其后，这个趋势不可逆。

有个值得想的地方：窗口期 = 端侧Agent从0到50%渗透这段时间。过了这个点，APP就算Agent化也只是端侧Agent的"可选后端"之一，不再是用户的智能伙伴。先发的意义不只是用户量，是**在端侧Agent接管之前建立直接的用户关系**。
