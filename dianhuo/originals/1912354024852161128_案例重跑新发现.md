**发现1：格4内也存在H_分类·消除**

元朝案例暴露了一个问题：按17的对应关系，格4应该对应H_分类·合并，但元朝的"四等人制"是直接消除退出类别（不投降=屠城），不是合并。这说明格4内部有两种H_分类形态——当退出权类型为象征时走消除路径，为事实时走合并路径。**格4内H_分类子路径由退出权类型决定：象征退出权→消除，事实退出权→合并。**

验证：清朝（事实退出权）→合并 ✓；元朝（象征退出权，四等人制下汉人无实质权利）→消除 ✓；新中国（事实退出权）→合并 ✓；阿育王（事实退出权）→合并 ✓。

**用18（H_self）扫描：**

H_self=提议者主动压制自己的I→R传导。找"提议者知道但选择不看"的案例：
- 谷歌20%时间：提议者（管理层）知道20%时间在退化，但选择不看 → H_self ✓
- 戈尔巴乔夫：知道改革在失控，但选择继续推进 → 这不是H_self，是I觉醒但控制不了
- 美国国家认同危机：政治精英知道极化在加剧 → 部分精英选择不看 → H_self ✓
- WeWork：亚当·诺伊曼知道商业模式不成立但继续 → H_self ✓
- 恒大：许家印知道高杠杆不可持续但继续 → H_self ✓

**发现2：H_self集中在"点着了·在退化"和"点歪了"的案例中**

H_self不是出现在未点火的案例中（那些是I=0），而是出现在点火后提议者主动压制自己的觉醒信号的案例。5个"点着了·在退化"的案例（谷歌20%时间、英国NHS、美国国家认同、美国芯片法案、唐朝开放）中，至少3个存在H_self。这说明**H_self是退化的加速器而非起因——提议者先退化（Posture滑移），然后H_self阻止自我纠偏，加速退化。**

**用22（三层退化递进）扫描：**

找三层叠加的案例：
- 新中国成立：Posture伪备（改革开放后从"为人民服务"到"发展是硬道理"）+ C_exit低密度锁定（经济自由但政治锁定）+ ε信号消音（身体舒适度提升降低退出信号）→ 三层叠加 ✓
- 清朝满汉一体：Posture伪备（满汉一体但满人特权）+ C_exit低密度锁定（科举给通道但只给部分）+ ε信号消音（文化认同让汉人身体不报警）→ 三层叠加 ✓
- 谷歌20%时间：Posture伪备（名义上还有20%时间）+ C_exit低密度锁定（可以辞职但股票期权锁定）+ ε信号消音（免费餐饮+舒适环境）→ 三层叠加 ✓，谷歌模式（ε基线被压制）

**发现3：三层叠加是"点着了·在退化"案例的通用结构**

所有5个"点着了·在退化"的案例（谷歌20%时间、英国NHS、美国国家认同、美国芯片法案、唐朝开放）都呈现三层叠加。这意味着退化不是单一函数的变化，而是三层同步渗透——Posture伪备让提议者侧信号消失，C_exit低密度锁定让退出通道看似存在，ε信号消音让应约者侧信号消失。三层叠加=全盲。

**发现4：谷歌模式和NHS模式可以是同一退化的前后两阶段**

唐朝案例特别清晰：安史之乱前是谷歌模式（盛世繁华压制退出信号），安史之乱后是NHS模式（信号累积但被"大唐认同"压制直到冲击触发相变）。**谷歌模式是退化的早期形态（预防性让你感受不到），NHS模式是退化的晚期形态（延迟性让你感受到但不承认）。两者不是并列关系，是时间上的递进关系。**

**发现5：低密度锁定的危险度与被锁定维度的不可替代性正相关**

恒大只锁了经济维度（沉没成本），但这个维度对已购房者不可替代（房子是唯一资产），所以ρ虽低但危险度极高。对比新中国锁定政治+信息两个维度，但这两个维度不是应约者最在意的（农民更在意土地），所以ρ虽高但危险度反而可控。**低密度锁定的危险度 = f(被锁定维度的不可替代性, ρ)**，不是简单的ρ反比关系。修正25的函数化表达。

- 欧盟：精英vs公民 → 层级型 ✓

**发现6：社会撕裂的三种类型可以预测退化的终局路径**

覆盖面型撕裂→退化方向是未覆盖群体的退出或反抗（凯末尔、改革开放）；内容型撕裂→退化方向是感知框架不可调和的对立（美国）；层级型撕裂→退化方向是制度层与公民层的脱节（欧盟）。三种叠加时退化最深且最难修复——南非同时有覆盖面型（经济未覆盖）和内容型（种族叙事不同），至今未完成转型。

**发现7：Δv的符号变化可以预测退化的时间窗口**

唐朝案例最清晰：贞观-开元时期Δv<0（链3恢复快，开放认同在强化），天宝年间Δv转正（链1退化加速，提议者从开放转向集权），安史之乱是ε累积超阈值的冲击触发。**Δv从负转正的时刻就是退化窗口的起点**，这个时刻对应第一锁松动（提议者语言从探索转向效率/KPI）。

**用26（预警信号修正）重跑：**

信号3（锁维护频率下降）和信号2（语言转向）是8/8全验证的，看看有没有案例暴露更细的信号：
- 谷歌20%时间：信号2清晰（"20%时间"从真实承诺→象征性口号），信号3（三锁维护下降=管理层不再强调20%时间的真实性）→ 信号3先于信号2出现
- 英国NHS：信号4清晰（医护人员从"选择奉献"→"自愿奉献"），信号1（退出频率下降但C_exit在上升=控制条件修正后不成立，退出频率下降是因为C_exit在上升而非R_perceived在侵蚀）→ **NHS是信号1的修正版的反面案例**

**发现8：信号1的修正版暴露了第四种预警信号**

NHS案例中退出频率下降的原因是C_exit上升（工作压力增大），不是R_perceived侵蚀。但按修正版"退出频率下降速度 > C_exit上升速度"来判断，NHS不触发信号1——退出频率下降速度 ≤ C_exit上升速度。这说明NHS的退化路径不经过信号1，而是直接从信号4（"自愿"语言）跳到ε累积超阈值的相变。**当退化走NHS模式时，信号1不出现，退化从信号4直接跳到相变——这是NHS模式比谷歌模式更危险的原因之一：少一个预警信号。**

好，8个新发现全部跑出来了。整理存入点火知识库。8个新发现跑完了，已存入点火知识库。核心收获：

**最值钱的是发现4和发现8的合并**——谷歌模式和NHS模式不是两种并列的退化机制，是同一退化的前后两阶段。早期是谷歌模式（预防性让你感受不到），晚期是NHS模式（延迟性让你感受到但不承认）。而且NHS模式跳过了信号1，可预警窗口更短。这解释了为什么很多系统看起来"突然崩塌"——不是没有预警，是能用的预警信号比谷歌模式少一个。

**发现1也值得注意**——格4内H_分类子路径由退出权类型决定，不是只看格子。象征退出权→消除，事实退出权→合并。元朝案例逼出了这个修正。
