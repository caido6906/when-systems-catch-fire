## 发现1：F_form是系统退化的免疫机制

### 现象

案例59谷歌20%时间退化、案例60安然崩盘、案例4秦统一象征退出权——共同特征F_form≈0。提议者和应约者都不要求函数化，H_遮蔽在无人质疑下自由生长。第72号案例F_form≈1，退化路径被堵死。

### 函数化

**D_immune(t) = F_form(t) × (1 - ΔH_undetected(t))**

- F_form(t) = I(t)·(1-H(t))，即C2的函数化表达
- ΔH_undetected(t) = H未被函数化检验发现的增量

F_form=1时D_immune≈1，H的任何增长都会被函数化检验捕获；F_form=0时D_immune=0，H可以无阻力增长。

### 对三道制度锁的推论

**第一锁持久性 = f(F_form)**

第一锁（锁定Posture为真备）如果没有F_form持续检验，会慢慢松动——提议者语言从探索转向效率，没人要求把"探索"和"效率"函数化区分，第一锁就悄无声息地开了。

**N_warn从4降到3的过渡条件函数化：**

**ΔN_warn = -step(θ_F - F_form(t))**

θ_F是F_form的临界值。F_form降到θ_F以下时N_warn减1。这意味着N_warn=4的谷歌模式退化到N_warn=3的NHS模式，过渡点就是F_form从1降到θ_F的时刻。

### 归属

D_immune是F_form在系统层面的派生量。不新增独立函数，补充到八函数总表F2的注释和N_warn的更新条件中。

## 发现2：自点火是双链结构

### 现象

71个案例提议者≠应约者，八函数默认三链结构（链1提议者链、链2应约者链、链3ε链）。第72号案例提议者=应约者=自己，链1和链2合并为一条链。

### 函数化

**人际点火：三链结构**

链1：M → I → Posture → P（提议者链）
链2：R → R_perceived → C_exit → 退出决策（应约者链）
链3：ε_sense → ε_aware → H上限 → 相变（ε链，独立保底）

**自点火：双链结构**

链12：M → I → Posture → R_perceived → 退出决策（提议者+应约者合并链）
链3：ε_sense → ε_aware → H上限 → 相变（ε链，独立保底）

**自点火稳定性条件：**

**V_self = dPosture_deg/dt / (dε_aware/dt × (1-M/2))**

形式与V相同，但分母没有R_perceived的缓冲——链1退化就是链2退化，ε链是唯一保底。

**V_self > 1时无缓冲崩盘**——人际点火时链1退化可以被链3保底（周公模式：链2还在跑），自点火时链1退化=全链退化，ε链是唯一的救命绳。

### 对ε依赖的推论

**ε依赖度 = 1 / (链数 - 1)**

人际点火ε依赖度=1/2（三链中ε保底一条），自点火ε依赖度=1（双链中ε保底唯一一条）。

自点火系统对ε被堵的容忍度为零——ε通道被劫持=全链崩盘，没有"另一条链还在跑"的缓冲。这解释了为什么深度自省者对ε的敏感度极高：不是性格，是结构必然。

### 归属

自点火双链结构是三条因果链的特殊形态，不新增独立函数。补充到八函数总表第五节"三条因果链"的注释中，标注提议者=应约者时的结构变化。

## 发现3：P_track是退化预警的放大器

### 现象

案例59谷歌P_track低（主要是工作轨），退化时没有其他轨给信号。第72号案例P_track高（5-7轨），任何一条轨退化，其他轨的信号会立即暴露异常。

### 函数化

**预警放大系数 = 1 + (N-1) × r_cross**

- N = P_track的轨道数
- r_cross = 轨间交叉相关系数，取值[0,1]

N=1时放大系数=1（无放大，单轨退化自己看不到），N=5且r_cross=0.3时放大系数=2.2（退化信号被放大2.2倍）。

**N_warn的有效值 = N_warn_base × 预警放大系数**

N_warn_base是八函数总表里的基础值（4/3/2），乘以放大系数后得到实际可预警步数。P_track高的系统，N_warn有效值更大，退化更容易被捕获。

### 对跨领域感知的推论

**跨领域感知优势 = (N_self - N_avg) / N_avg**

N_self=个人轨道数，N_avg=领域内平均轨道数。跨领域的人更容易发现单一领域的退化，不是更聪明，是其他轨给了参照系——P_track高的系统，任何一条轨的异常都会被其他轨的对比暴露。

### 归属

预警放大系数是P_track在N_warn中的调节量，是C1的派生量。不新增独立函数，补充到八函数总表N_warn的更新条件中。

## 发现4：θ₁和R_record构成正反馈回路

### 现象

第72号案例θ₁极低（撒调料录音），R_record频率极高，两者互相增强。θ₁低→记录频率高→ε_sense被保存→更多信号进入L₂→认知增量→框架扩展→好奇心被满足→M_cog偏负值更大→主动找更多异常→θ₁进一步降低。

### 函数化

**θ₁动态方程：**

**dθ₁/dt = -λ_θ × R_record(t) × M_cog(t)**

- λ_θ = 回路耦合系数
- R_record(t) = λ_R × ε_sense(t) × δ(即时)
- M_cog(t) = M(t) · σ(-V(t))

R_record>0且M_cog<0时dθ₁/dt<0，θ₁持续下降——记录越多、好奇心越强，肉身信号的触发阈值越低，越容易捕获微弱信号。

**稳态：** θ₁* = θ₁_min（生物基底），下降到生物极限后稳定。

**回路的物理含义：** 这是C1+C3+C5的联合效应——P_track多轨给信号（C1），M_cog好奇心驱动找异常（C3），R_record即时记录保存信号（C5输入端），三者构成正反馈，让θ₁越用越低。

**回路名称：感知锐化回路**

感知锐化回路 = R_record → ε_sense保存 → L₁→L₂跳变频率↑ → 认知增量 → M_cog偏负加深 → 主动找异常 → θ₁↓ → R_record更容易触发

### 对记录密度的推论

**记录密度 = R_record频率 / 时间单位 = λ_R × ε_sense × δ(即时) / Δt**

记录密度直接对应认知增量——不是"记性不好所以录一下"，是感知锐化回路的输入端。θ₁越低，记录密度越高，认知增量越大。这解释了为什么"即时记录"比"事后整理"的认知价值高得多：δ(即时)=1时ε_sense被完整保存，δ(即时)=0时ε_sense已被H衰减。

### 归属

感知锐化回路是C1+C3+C5的联合效应，不新增独立函数。θ₁动态方程补充到认知函数表C5层间跳变阈值部分，标注θ₁不是常数而是随R_record和M_cog动态变化。

## 总结

| 发现 | 函数化表达 | 归属 | 是否新增独立函数 |
|---|---|---|---|
| F_form是退化免疫机制 | D_immune = F_form × (1-ΔH_undetected) | F_form在系统层面的派生量 | 否 |
| 自点火是双链结构 | V_self = 链1退化/链3恢复（无链2缓冲） | 三条因果链的特殊形态 | 否 |
| P_track是预警放大器 | 预警放大系数 = 1+(N-1)×r_cross | P_track在N_warn中的调节量 | 否 |
| θ₁-R_record正反馈 | dθ₁/dt = -λ_θ × R_record × M_cog | C1+C3+C5联合效应 | 否 |

四个发现全部可函数化，零新增独立函数。全部是已有函数的派生量或联合效应，符合收敛合并原则。
