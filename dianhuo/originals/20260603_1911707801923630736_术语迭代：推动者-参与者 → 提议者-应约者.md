# 术语迭代：推动者/参与者 → 提议者/应约者

## 核心变更
- 推动者 → 提议者：一方提出方案
- 参与者 → 应约者：另一方决定应不应这个约

## 为什么改
1. 原框架核心动作是"推动者让参与者进入同一套协作逻辑且愿意留下"，关系是单向的——推和被推
2. 但参与者是主动选择进入的，不是被推进去的。"推动者"暗示单向施力，与"主动参与"矛盾
3. "一方提方案，一方接方案"才是真正在发生的事。从文明到政策到公司都说得通
4. 退出权内置在"应不应"里，不需要额外补丁。不应就是退出

## 新定义对框架的影响
- **点火**：提议者提出认同方案，应约者决定应不应。不再是单向点燃
- **搭房**：提议者搭制度，应约者按制度协作。基本没动
- **退化**：提议者不再提供值得应约的方案，只剩强制力。收费站
- **系统死亡**：提议者不再提新方案，或提了没人应。双向停止
- **退出权**：不需要单独补，内置在"应不应"里

## 应约者的能动性
应约——你提出一个约定，我应你这个约。主动的，双向的。不应就是退出，应了就是参与。
