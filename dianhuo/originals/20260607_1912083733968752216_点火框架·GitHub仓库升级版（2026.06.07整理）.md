# 点火框架·GitHub仓库升级版（2026.06.07整理）

## 一、项目定位

不是简单托管在GitHub上的书稿，而是一个 Agent-readable book project——同时面向人类读者和AI Agent读者的双读者知识工程。人类通过 book/ 和 SUMMARY.md 进入正文；AI Agent 通过 llms.txt、llms-full.txt、agent/ 和 data/ 快速读取概念、论点、案例和证据。

仓库四层结构：整理层（book/）→ 来源层（dianhuo/）→ AI交互层（agent/）→ 数据层（data/）

许可协议：CC BY-NC-SA 4.0（署名—非商业性使用—相同方式共享）

## 二、核心定义升级

旧版：系统 = 协作系统 + 认同系统
新版：事件 = 提议者出方案 + 应约者接或不接

系统不是事件之外的东西。系统是同一个事件被足够多人、足够长时间地重复，直到它看起来像固体。点火框架拆的不是静态制度，而是事件走向。

核心术语：提议者（一方提方案）/ 应约者（一方接或不接）。退出权内置在"应不应"里。

## 三、事件的最小结构

每个事件包含三个不可再分的时刻：
1. 提议：有人把方案摆出来
2. 应约：有人接了，或没接
3. 走向：接了之后往哪走，没接之后怎样

## 四、事件的刻度（新增：二维尺度矩阵）

旧版：文明/国家/政策/公司/亲子 五级分类
新版：事件尺度 = 参与者规模 × 持续时间，旧标签只是常见区域的简写

参与者规模（P1-P6）：双人→小群体→组织→群体→国家人口→文明人口
持续时间（T1-T6）：瞬时→短期→年度→十年级→世纪级→千年级

刻度不同，骨架相同：提议者出方案，应约者接或不接，退出权在不在，叙事权归谁。

## 五、条件层重构：24格→8格

旧版24格：提议者意识×提议者退出权×应约者退出权×应约者状态=24格
新版8格：只保留纯输入变量，应约者状态移至方向层

| 格 | 提议者意识 | 提议者退出权 | 应约者退出权 |
|---|---|---|---|
| 格1 | 有 | 有 | 有 |
| 格2 | 有 | 有 | 无 |
| 格3 | 有 | 无 | 有 |
| 格4 | 有 | 无 | 无 |
| 格5 | 无 | 有 | 有 |
| 格6 | 无 | 有 | 无 |
| 格7 | 无 | 无 | 有 |
| 格8 | 无 | 无 | 无 |

8格×3种应约状态=24种组合，信息量没丢，但因果逻辑干净了——输入是输入，输出是输出。

## 六、五步事件走向

1. 旧事件失灵：上一轮方案不再值得应约
2. 提议者出现：有人提出新方案
3. 应约者应约：有人接了，或没接
4. 点火跑通：应约者不仅接了，还认了
5. 稳定期：事件固化成系统，等下一轮失灵

制度是凝固了的事件。

## 七、提议者的两种姿态（新增）

推方案（培养型/僭越）：替应约者决定需要什么，把方案推过去。"应"不是应约，是被推着走。
备方案（培育型/管家）：方案备着，等应约者需要时自己来拿。只有备方案模式下的"应"才是真应约。

推方案是僭越，备方案是管家。

## 八、五种事件结果

点对了：认同建在应约者身上，事件健康运转
点不足：只覆盖部分应约者，事件半活半死
点歪了：认同建在外部参照上，退化信号被叙事权遮蔽
点反了：灭了旧认同但没建新认同，比之前更糟
负时间差：先拆旧的再建新的，真空期足以杀死事件

## 九、退出权四分法（新增）

1. 真实退出：人走了，系统留不住
2. 事实退出：人还在，但不参与核心协作
3. 心理退出：身体还在系统里，认同已经撤回（最隐蔽也最危险的退化信号）
4. 象征退出：仪式性表态，不改变实质

## 十、事件如何死亡（新增）

事件通常不是被外力摧毁，而是从内部停止迭代。提议者退化四条路径：
1. 信息回路断裂：不知道该提什么
2. 激励偏移：不想提了
3. 权力腐蚀：不需要提了
4. 组合型：多条路径叠加，加速退化

## 十一、多柱互撑事件（新增）

有些事件不是一根柱子支撑，而是几根柱子互相锁定。一个柱子的退化信号会被其他柱子遮蔽，退出这个念头本身可能被文化编码提前消解。比单柱强制系统更稳定，也更难被识别为退化。

## 十二、叙事权垄断（新增）

点歪了不容易被识别，因为评价标准本身可能是提议者写的。应约者用提议者的尺子量自己，退化信号被叙事权遮蔽。

## 十三、框架边界

只拆"方案输出—承接"型事件。不覆盖：纯外部冲击（瘟疫、天灾、外部征服）、收费站型结构（只收租不提供应约价值）、方案本身造假的结构。
