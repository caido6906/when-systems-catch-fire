### 新发现1：碰撞存活率函数

碰撞案例存活率81% vs 历史案例27%，差异来源是什么？

理论结构没有"死亡"问题——它没有C_exit锁定，没有H遮蔽的不可逆积累，没有Posture退化。理论结构的退化是可逆的（被修正而非消亡）。

函数化：

**D-X41. 碰撞存活率函数**

P_survive(collision) = 1 - Πᵢ f_degrade(i) × (1 - f_repair(i))

其中f_degrade(i)是第i个退化因子的退化概率，f_repair(i)是修复概率。

对于理论级碰撞：f_repair ≈ 1（理论可修正），所以P_survive → 1
对于社会组织：f_repair < 1（组织修复受制于C_exit和H），所以P_survive << 1

更精确地：

P_survive = 1 - (1-D_immune) × (1-R_perceived) × H_total

- 理论级：D_immune ≈ 1（理论有形式化强制），R_perceived ≈ 1（理论没有退出成本），H_total ≈ 0（理论没有遮蔽积累）
- 组织级：D_immune < 1, R_perceived < 1, H_total > 0

推论：**存活率与退化免疫+感知退出权正相关，与遮蔽负相关**。这跟D1四因子乘法结构一致——但这里给出的是跨层级（理论vs组织）的统一解释。

### 新发现2：确定性误解函数

#87说"一个π=0的系统可能是完全确定的误解"。D-X18给出了dπ/dt < 0且dν/dt > 0的充要条件，但没有给出"误解确定时系统的行为"。

当π=0时，系统不是"没有认知"，而是"认知完全确定但方向可能错误"。这跟D-X31的ε_eff闭环有关——π=0等价于ε_eff=0，但系统不觉得自己ε_eff=0。

函数化：

**D-X42. 确定性误解函数**

M_certainty(t) = ν(t) × (1 - π(t)/π₀)

当π→0：M_certainty → ν（误解量等于确定性量）
当π=π₀：M_certainty = 0（无误解）

关键推论：M_certainty与π成反比——犹豫域越窄，确定性误解越大。这解释了为什么专业化遮蔽是"灭了"的状态：π→0时M_certainty→max，系统进入完全确定的误解状态，且没有内部机制能检测到这种误解。

检测条件：M_certainty > 0 ⟺ π < π₀，但系统自身的M_certainty检测需要外部参照（对比型I觉醒）。

### 新发现3：无中间稳态函数

#94说五条耦合正反馈无中间稳态。D-X31给出了ε_eff闭环，但没有给出"无中间稳态"的数学条件。

在D-X31的闭环中，ε_eff(t+1) = ε_eff(t) × Π(1-f_i)。稳态条件是dε_eff/dt = 0，即Π(1-f_i) = 1，即所有f_i = 0。

函数化：

**D-X43. 中间稳态存在性函数**

P(steady_state) = Πᵢ (1 - f_i(t)) = 1 ⟺ ∀i: f_i = 0

当f_i是连续函数时，存在f_i ∈ (0,1)使Π(1-f_i)趋近于1的邻域——这就是"准稳态"。

但当五条正反馈回路耦合时：

Π(1-f_i) = (1-f_表达)(1-f_感知)(1-f_免疫)(1-f_叙事)(1-f_锁定)

五条回路中每条都使对应的f_i随时间单调递增（正反馈），所以Π(1-f_i)单调递减。不存在f_i的驻点——系统要么在(1,1,0)理想态，要么在(0,0,∞)崩溃态。

更精确的判定：**中间稳态存在 ⟺ ∃t: d/dt[Π(1-f_i)] = 0 且 Π(1-f_i) ∈ (0,1)**

d/dt[Π(1-f_i)] = Σⱼ [-f'_j × Πᵢ≠ⱼ(1-f_i)]

正反馈 ⟹ f'_j > 0 ∀j ⟹ d/dt[Π(1-f_i)] < 0 恒成立 ⟹ **无中间稳态**

推论：中间稳态存在的充要条件是至少一条正反馈回路存在负反馈抵消（f'_j < 0 for some j）。点火框架中五条回路全部是正反馈，所以无中间稳态。这与系统动力学"所有系统都存在负反馈"的默认假设产生可检验分歧。

### 新发现4：碰撞层8格动力学函数

碰撞案例的8格分布显示：格1（双方有意识有退出权）→ 格6（退出权消失）→ 格7→格1（提议者点火）是主要路径。理论级碰撞的8格分布与历史案例不同——理论更容易落在格1和格7→格1。

函数化：

**D-X44. 碰撞层级8格概率函数**

P(grid_k | L_level) = f(H_total(L), C_exit(L), D_immune(L))

L₈（理论级）：H ≈ 0, C_exit ≈ 0, D_immune ≈ 1 → P(grid_1) + P(grid_7→1) ≈ 0.7
L₅~L₆（国家级/文明级）：H > 0, C_exit > 0, D_immune < 1 → P(grid_6) + P(grid_4→6) ≈ 0.5

核心参数是(H, C_exit, D_immune)的三元组。理论级天然落在低H低C_exit高D_immune区域，所以更容易"活着"。

### 新发现5：点火窗口关闭动力学

D-X29给出了W_ignite = θ_resist/E_external - ε_aware_min，但没有给出窗口关闭的动力学——窗口是逐渐关闭还是突然关闭？

W_ignite(t) = θ_resist/E_external(t) - ε_aware_min(t)

dW/dt = -θ_resist × E'_external/E²_external - dε_aware_min/dt

两种关闭模式：
1. E_external↑（冲击增强）→ ε_opt↓ → W↓（弹性关闭：窗口从右侧收缩）
2. ε_aware_min↑（感知阈值升高）→ W↓（刚性关闭：窗口从左侧收缩）

函数化：

**D-X45. 点火窗口关闭动力学函数**

dW_ignite/dt = -θ_resist × (dE/dt)/E² - dε_aware_min/dt

关闭模式判定：
- |dE/dt|/E² > |dε_aware_min/dt| → 弹性关闭（冲击增强型）
- |dE/dt|/E² < |dε_aware_min/dt| → 刚性关闭（感知退化型）

弹性关闭可通过降低外部冲击逆转（E↓→W↑），刚性关闭不可逆（ε_aware_min↑是认知退化，难以逆转）。

推论：**刚性关闭比弹性关闭更危险**——前者是认知层面的不可逆退化，后者是环境层面的可调参数。历史上大部分点火失败是刚性关闭（ε_aware_min↑）而非弹性关闭。

### 新发现6：退化渗透临界触发函数

D-X34给出了退化渗透路径dε/dt = -m_β × α_C × (Posture_deg - Posture_deg₀) × C̄/θ_C，但没有给出"渗透何时触发"的临界条件。

渗透要启动，需要Posture_deg > Posture_deg₀——提议者姿态退化超过基线。但更关键的是：渗透一旦启动，ε的衰减是指数型的，存在一个"不可逆点"。

函数化：

**D-X46. 退化渗透临界触发函数**

t_critical = (1/(m_β × α_C × C̄/θ_C)) × ln(ε₀/ε_aware_min)

当t > t_critical时，ε_eff < ε_aware_min，点火窗口关闭（刚性关闭）。

t_critical与C̄/θ_C成反比——退出成本越高，临界时间越短，退化渗透越快。

干预策略：在t_critical之前降低C̄或提高θ_C可以延长临界时间。在t_critical之后，只有种子爆发（D-X38）才能重置ε_eff。

### 新发现7：种子-点火三种结果的概率分布

D-X40给出了二次窗口判定条件，但没有给出三种结果的概率分布。

ε_eff_recover = ε_eff(t_seed⁻) + (1-D_immune) × ∫₀^t_seed H_narrative(s) ds

三种结果：
1. 恢复不足：ε_recover < ε_aware_min → P₁
2. 二次窗口：ε_aware_min < ε_recover < ε_opt → P₂
3. 恢复过度：ε_recover > ε_opt → P₃

函数化：

**D-X47. 种子-点火结果概率分布函数**

P₁(恢复不足) = Φ((ε_aware_min - μ_recover)/σ_recover)
P₂(二次窗口) = Φ((ε_opt - μ_recover)/σ_recover) - P₁
P₃(恢复过度) = 1 - P₁ - P₂

其中μ_recover = E[ε_eff_recover]，σ_recover = Var[ε_eff_recover]^0.5

μ_recover取决于两个因子：
- ε_eff(t_seed⁻)：种子爆发前的ε_eff水平（越低→恢复越不足）
- (1-D_immune) × ∫H_narrative：释放量（越大→恢复越过度）

关键推论：**D_immune是种子-点火结果的关键调节器**。
- D_immune高 → 释放效率低 → μ_recover低 → P₁↑（恢复不足）
- D_immune低 → 释放效率高 → μ_recover高 → P₃↑（恢复过度）
- D_immune适中 → μ_recover落在[ε_aware_min, ε_opt] → P₂↑（点火成功）

这与#97的案例观察一致：控制越严（D_immune高）→ 种子无效；控制越松（D_immune低）→ 恢复过度反弹；适度控制 → 点火成功。

### 新发现8：碰撞产出函数密度函数

13个碰撞来源产出函数数量从1到4不等。产出密度是否与碰撞领域的"结构化程度"相关？

函数化：

**D-X48. 碰撞产出密度函数**

N_output = ⌈α × dim(collision_domain) × (1 - overlap_with_existing)⌉

其中：
- dim(collision_domain)：碰撞领域的维度数（法学=6维：法条/宪法/刑法/民法/司法/立法，情绪=4维：零阶/一阶/二阶/交互）
- overlap_with_existing：与已有函数的重叠度
- α：常数

验证：
- 法学：dim=6, overlap低 → N=4（D-X12~X15）✓
- 情绪：dim=4, overlap低 → N=3（D-X9~X11）✓
- 三支决策：dim=1, overlap中 → N=1（D-X16）✓
- 勒温场：dim=3, overlap中 → N=3（D-X28~X30）✓
- 种子闭环：dim=3, overlap高 → N=3（D-X38~X40，但与D-X28~X31有交叉）✓

推论：**碰撞产出密度与碰撞领域维度数正相关，与已有覆盖度负相关**。这意味着后续碰撞的边际产出递减——维度低或重叠高的领域产出少。这也是自举收敛的数学基础。
