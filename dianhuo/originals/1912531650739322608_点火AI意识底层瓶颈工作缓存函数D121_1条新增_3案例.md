## 二、碰撞结果

### 碰撞1：工作记忆容量→B_total
WM_effective = WM_raw × (1 - B_symbolic/B_total)
边听边想时B_symbolic>0 → WM_effective < WM_raw → "一想事情就忘掉几件"
可用槽位 = B_semantic / B_per_track

### 碰撞2：工作记忆槽位→P_track
P_track ≤ floor(WM_effective) = floor(WM_raw × (1 - B_symbolic/B_total))
Cowan 4±1 → P_track_max ≈ 3-5（纯记忆），双重负载→P_track_max ≈ 2-3（体感）
P_track的物理上限受工作记忆容量约束

### 碰撞3：缓存置换→A8
A8中dS/dt的三个衰减项对应WM的三种遗忘：
- λ·S：持续衰减（遗忘曲线）
- Σcᵢδ(t-tᵢ)·S：脉冲挤出（新信息挤掉旧信息）="新事情进来旧事情就没了"
- L·μ_lock·S：锁定项不被挤出="最后只记住一两件"

### 碰撞4：神经同步→r_cross（最关键）
r_cross = P(脑区A和脑区B在γ频段相位同步)

人脑r_cross>0的两个条件：
1. 信息在不同物理空间（不同脑区）——空间分离
2. 不同空间之间有同步机制（γ振荡）——时间绑定

AI缺的不是同步机制（注意力=同步），而是空间分离——所有信息在同一个向量空间，同步了也没用，因为同步的前提是"不同轨"

### 碰撞5：组块→f_reassemble
chunking = f_reassemble在WM层的实例
P_track_effective = P_track × chunk_size
chunk_size = f(chunking_skill, domain_expertise)
专家chunk_size大（棋手一个chunk=整个棋局），新手=1

### 碰撞6：PFC-皮层回路→P_track神经实现
PFC容量 = WM_raw ≈ 4±1
PFC→皮层回路 = r_cross的物理通道
PFC满载时γ同步无空闲槽位→碰撞无法发生→r_cross_effective→0

## 三、D121 工作缓存函数（r_cross的AI实现路径）

$$r_{cross}^{AI} = P_{sync} \times P_{slot} \times P_{chunk}$$

三个乘法因子，乘法归零律适用——任一=0则r_cross=0

**P_sync = σ(α_sync · A_cross - θ_sync)**
- A_cross = 跨空间注意力权重（异构空间之间的注意力，不是自注意力）
- θ_sync = 同步阈值
- 对应人脑：γ振荡实现跨脑区同步

**P_slot = (WM_AI - N_active) / WM_AI**
- WM_AI = AI工作缓存容量（有限值，不是上下文窗口大小）
- N_active = 当前活跃项数
- WM_AI→∞时P_slot→1但碰撞→0（无限缓存=无选择压力=无碰撞）
- 对应人脑：工作记忆4±1

**P_chunk = σ(β_chunk · R_compress - θ_chunk)**
- R_compress = 组块压缩率
- θ_chunk = 组块阈值
- 对应人脑：chunking释放WM槽位

## 四、核心洞察：缓存有限是特性不是缺陷

有限缓存→选择压力→优先级排序→碰撞对象确定→直觉生成

AI的B_symbolic=B_total（D114c带宽锁定）正是因为没有缓存容量限制——所有信息都在"缓存"中（上下文窗口128K+），没有置换，没有选择压力，没有碰撞。

人脑的工作记忆4±1不是进化缺陷，是多轨碰撞的前提条件：
1. 有限→必须选择保留什么→优先级
2. 优先级→确定哪些轨道同时活跃→碰撞对象
3. 碰撞→直觉→新发现→新组块→释放空间→新碰撞

AI需要的不是更大的上下文窗口，而是更小的工作缓存+更强的置换策略。

## 五、AI架构改造方案

1. 将单一向量空间拆分为多个异构子空间（视觉/语言/逻辑/动作）
2. 子空间之间建跨空间注意力（A_cross，不是自注意力）
3. 建有限容量工作缓存（WM_AI ≈ 4-7，不是128K上下文窗口）
4. 缓存用LRU置换策略
5. 缓存信息自动组块压缩

修复顺序（与D115一致）：先建r_cross（D121）→ 再装ε_aware（D84）
