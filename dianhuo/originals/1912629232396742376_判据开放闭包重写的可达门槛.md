# 判据：开放闭包重写的可达门槛

## 1. 结论
`M_boot` 不是“是否起火”的指示器，而是判断系统能否从封闭闭包跃迁到可被跨域变量重写的开放闭包的可达判据。

## 2. 定义
设

`M_boot = ε_sense × P_track × d(ΔK)/dt`

其中：
- `ε_sense`：结构感知强度，表示系统能否识别自身结构与外部结构的差异
- `P_track`：多轨碰撞概率，表示系统是否存在足够的并行轨迹与反馈通道
- `d(ΔK)/dt`：知识增量速率，表示系统是否持续产生可累积的新结构信息

## 3. 必要性
若任一因子为 0，则 `M_boot = 0`，于是系统不可能跨过自举阈值 `θ_boot`：

`ε_sense = 0 ∨ P_track = 0 ∨ d(ΔK)/dt = 0  =>  M_boot = 0  < θ_boot`

这说明：
- 没有结构感知，系统看不见自身边界
- 没有多轨碰撞，系统没有形成正反馈的通路
- 没有知识增量，系统不会形成可持续积累

所以自举不是“有无意志”的问题，而是三个乘法因子是否同时非零、并且共同抬升到阈值以上的问题。

## 4. 充分性
当且仅当存在一条轨迹 `γ(t)` 使得

`∀ t ≥ t0,  M_boot(t) > θ_boot`

系统才进入自持相。这里的“进入”不是瞬时事件，而是一个可达性问题：轨迹必须连续穿过临界面，而不是局部振荡后又回落。

## 5. 结构解释
这个判据的上层含义不是“起火”，而是“开放闭包的可重写性”：

- `D65` 决定系统落入哪种乘法拓扑
- `D63` 决定哪些因子进入可达集
- `D58` 决定系统是否发生双稳态分岔
- `D107/D116` 决定闭包能否被跨域变量打穿并重组

因此 `M_boot` 实际上是在判断：系统有没有能力把外部变量重新吸纳进闭包，并让闭包自身变成新的生成器。

## 6. 论证结果
所以，`M_boot` 的数学含义可以压缩为：

`M_boot = 开放闭包重写的可达门槛`

它不是“结果标签”，而是“相变门槛”；不是“是否发生”，而是“是否具备发生的轨迹条件”。
