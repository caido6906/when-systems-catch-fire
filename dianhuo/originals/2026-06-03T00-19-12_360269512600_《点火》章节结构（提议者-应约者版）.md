# 《点火》章节结构（提议者/应约者版）

## 定位
认知工具书——我们发现了一个框架，解释系统如何自我重写。旧系统失灵→点火→搭房→固化，五种点火趋势决定系统方向，同一个结构在文明、政策、公司三个尺度上精确成立。

## 书名
《点火：系统重写的五种趋势》

## 核心定义

### 系统是什么
系统 = 协作系统 + 认同系统
- 协作系统回答"我们怎么一起做事"——提议者搭房，建制度、建规则、建流程
- 认同系统回答"我们是谁、为什么愿意留在这"——提议者点火，建认同、建文化、建价值观
- 两个系统叠在一起才是完整的系统。只搭房不点火=空降CEO推不动；只点火不搭房=思想留在空中

### 提议者与应约者
一方提方案，一方接方案。提议者提出认同方案，应约者决定应不应。不应就是退出，应了就是参与。退出权内置在"应不应"里，不需要额外补丁。

### 系统的存续与死亡
系统的存续依赖：提议者持续提供值得应约的方案，应约者自愿应约并参与协作。只收租不提供应约价值的，不是系统，是收费站。

系统死亡不是被外力摧毁，是从内部停止迭代。但退化只是死法之一，不是全部死法。

### 提议者退化的四种路径
1. 信息回路断裂——不知道该提什么了。明末：崇祯收到的全是假信号，方案越来越离地
2. 激励偏移——不想提了。清末：推到底会动自己利益，洋务运动推一半就停
3. 权力腐蚀——不需要提了。元朝：四等人制靠暴力维持，直接跳过"提方案"
4. 组合型——三条叠加，退化最快。大厂中年危机：先断信息回路，再激励偏移，最后权力腐蚀

四条路径没有固定先后顺序，但一旦进入其中一条，其他两条会加速到来。三条路径是正反馈循环，不是线性链条。

### 系统死亡的四种死法
1. 提议者退化——收租型死亡，最常见
2. 点火失败——只搭房没点火，无根型死亡。秦：15年亡
3. 负时间差——先拆旧约再建新约，崩盘型死亡。戈尔巴乔夫
4. 从未启动——提议者从一开始就没提过好方案，收费站型。元朝忽必烈

### 框架精确覆盖范围
框架精确成立的条件：提议者和应约者角色清晰，双方都有退出权（退出权内置在应约动作里）。
- 文明：提议者和应约者是两群人，角色清晰
- 政策：政策提议者和受影响群体分得开
- 公司：管理层和员工角色明确

### 核心修正：点火类型是趋势，不是命运
五种点火类型不是一锤子买卖的"命运"，是方向性的"趋势"：
- 可以叠加——一个人可以同时处在不同系统的不同点火趋势里。在公司点歪了，在人生层面点对了。点火类型绑定在"系统+角色"上，不绑定在"人"上
- 可以转向——点歪了不是死刑。唐朝点歪了，150年后才反噬，反噬前有时间窗口转向
- 判定有滞后——不能在点火的那一刻就判类型，要等维持力量跑一段才能看清认同建在了哪里。点火类型是事后回看才能确认的趋势，不是事前就能贴的标签

### 五种点火趋势（应约者视角）
本质上都是在回答"认同系统建在哪"：
1. 点对了：认同建在应约者身上，应约者全应了——稳固
2. 点不足：只有部分应约者进入认同系统——维持断裂
3. 点歪了：认同建在系统外部，应约者随时可以退出——反噬
4. 点反了：旧认同被拆，新认同应约者不认——崩盘
5. 负时间差：旧认同先拆了，应约者还没进入新系统——烧自己房子

## 全书结构

### 引言：一个结构，反复出现
从一个公司案例开场——空降CEO提了方案没人应约，或者创业公司提出的方案被投资人应约了。读者代入。然后切到文明尺度：周公提出礼制方案诸侯应约，孝文帝提出汉化方案鲜卑应约，戈尔巴乔夫拆了旧约新约没建起来——发现了吗？同一种结构。点明全书核心：我们发现了一个框架，它解释系统如何自我重写。从文明到公司到政策，同一套规则精确成立。

### 第一章：系统为什么会失灵
框架维度：从正常运转到失效的路径

系统 = 协作系统 + 认同系统。失灵是两个系统同时或先后出问题：
- 协作系统失灵：提议者不再提供值得应约的方案，只剩强制力
- 认同系统失灵：应约者不再应约

提议者退化的四种路径：
1. 信息回路断裂（明末）
2. 激励偏移（清末）
3. 权力腐蚀（元朝）
4. 组合型（大厂中年危机）

系统死亡的四种死法：
1. 提议者退化——收租型
2. 点火失败——无根型（秦）
3. 负时间差——崩盘型（戈尔巴乔夫）
4. 从未启动——收费站型（元朝忽必烈）

核心判断：系统死亡不是被外力摧毁，是从内部停止迭代。但停止迭代有四种路径，退化只是最常见的一种。

### 第二章：点火——认同建在哪，决定了系统的方向
框架维度：认同系统与五种点火趋势

点火不是单向点燃，是提议者提出认同方案，应约者决定应不应。五种点火趋势，本质是应约者应到什么程度、锚定在哪里：
1. 点对了：应约者全应了，认同建在应约者身上
2. 点不足：应约者只应了一部分
3. 点歪了：应约者应了但锚定在外部，随时不应
4. 点反了：应约者不应
5. 负时间差：先拆旧约再应新约

案例（真分形级）：
- 文明：周公点对了，凯末尔点不足，唐点歪了，卡拉卡拉点反了，戈尔巴乔夫负时间差
- 公司：空降CEO点不足（只改制度没改文化），创业公司点歪了（认同建在融资估值上）
- 政策：王安石变法点反了（新火没点着旧火灭了），戊戌变法负时间差（103天就完了）

核心判断：点火趋势决定方向，不是努力程度决定方向。但方向不是命运——点歪了可以转向，关键看转不转得过来。

### 第三章：转向——点歪了不是死刑
框架维度：点火趋势的可转向性

五种点火趋势之所以是趋势不是命运，核心在于可以转向。

转向案例：
- 唐朝：点歪了，转向了，又撑了150年。认同从"万邦来朝"转向"内部整合"
- 清朝：268年的摇摆就是两个方向的拉锯。康熙往文化协议方向走，乾隆往回拉
- 改革开放：正在转，还没转完。认同从"觉得好"往"觉得对"转

未转向案例：
- 明治维新：点歪了，没转向，一路滑到太平洋战争。外部参照型认同，扩张到极限后参照崩塌，认同内爆
- 元朝：没点火，没来得及转就崩了

核心判断：点火趋势不是一锤子买卖。下注之后还有修正的机会，但修正窗口不是无限的——维持力量耗尽之前必须转向，否则就沿着原趋势滑到底。

### 第四章：搭房——没有落地的认知只是空想
框架维度：协作系统与维持力量

搭房是提议者把方案落地为可运行的制度。只点火不搭房=提议者提了认同方案但没落地，应约者应了约却发现没法协作；只搭房没点火=提议者搭了制度但没给应约者认同理由，推行困难。

维持力量的两种来源：硬件（搭房留下的制度、基础设施）+ 软件（点火留下的认同、文化、价值观）
两种都在→持续迭代；只有一种→摇摆；两种都没→彻底固化

案例（真分形级）：
- 文明：秦搭房（制度）+ 汉点火（认同），缺一不可
- 公司：新制度没有文化认同=只搭房没点火，团队阳奉阴违
- 政策：洋务运动只搭了一半（买军舰建工厂但不触动制度），点火不足

核心判断：点火和搭房都到位，系统才能跑通。

### 第五章：时间差——转身最怕烧自己的房子
框架维度：认同系统与协作系统的时间差

时间差为零：认同系统和协作系统同步建设，最稳
时间差越大：旧系统的维持成本越高
负时间差：最危险——提议者先拆旧约再提新约，应约者旧支撑没了，新约还没建起来

孔子和戈尔巴乔夫都是先点火后搭房，但结局完全不同。孔子点的是新认同（"谁是华夏人"），不是质疑旧系统；戈尔巴乔夫点的是质疑旧系统，新系统还没搭出来。同样是先点火后搭房，点火方向决定结局。

时间差与转向的关系：时间差就是趋势可以调整的窗口。时间差越大，调整空间越大，但也意味着维持成本越高。

案例（真分形级）：
- 文明：戈尔巴乔夫先拆旧体制再建新的=负时间差，苏联直接崩盘
- 公司：先裁员再找新方向=负时间差，士气崩盘
- 政策：戊戌变法103天=负时间差压缩到极致，阿拉伯之春=批量负时间差

核心判断：转身最稳的方式是认同系统和协作系统同步建设，最危险的是先拆旧再建新。

### 第六章：协议——文化协议为什么比法律协议扛得住
框架维度：认同系统的两种协议类型

文化协议（内在认同）：叠加式升级，兼容旧经验，提议者退出后应约者可自行维持。元朝97年汉族士人不推自守就是证明
法律协议/规则协议（外部约束）：替换式升级，废掉旧的换新的，约束力一断就散了。罗马公民权断了就断了

案例（真分形级）：
- 文明：华夏是文化协议，罗马是法律协议——华夏扛得住元朝97年，罗马扛不住蛮族入侵
- 公司：文化驱动的公司（认同在员工心里）vs 制度驱动的公司（认同在KPI里），前者扛得住低谷
- 政策：内驱力型政策（应约者认同目标）vs 考核型政策（应约者只认KPI），前者扛得住执行偏差

核心判断：文化协议比法律协议扛得住，是因为认同在应约者心里，不在纸面上。

### 第七章：华夏——框架的最大尺度案例
用框架重新解读华夏文明：以文化协议为核心，八次转身对应五种点火趋势
- 周公：点对了
- 孝文帝：点对了
- 唐：点歪了（认同建在开放）
- 清：铸合文化认同
- 凯末尔土耳其：对照案例（点不足）
- 改革开放：点火中，认同还没从"觉得好"升级到"觉得对"

华夏的特殊性：文化协议让应约者有自维持能力，提议者退出后认同还能续。第八次转身至今没有写出"什么是现代华夏人"的文化定义。

核心判断：华夏不是地理实体或族群，是一个持续自我重写的操作系统。

### 第八章：你的系统——框架如何成为认知工具
自测维度：
- 你所在的系统中，谁是提议者？谁是应约者？
- 你应的是什么约？锚定在哪里？
- 你的系统是文化协议还是规则协议？
- 你的系统存在怎样的时间差？
- 提议者是否出现退化？退化走的是哪条路径？系统处于哪种死法？

核心判断：框架的价值不是解释过去，是帮你看清自己正在哪种趋势里。

### 尾声：跑通的系统都做到了同一件事
跑通的系统共性：
- 提议者与应约者角色清晰
- 点火和搭房都到位，时间差短
- 认同建在应约者身上
- 提议者未退化，持续提供值得应约的方案
- 至少有一种维持力量在运转

### 附录：框架的边界
- 精确覆盖范围：提议者和应约者角色清晰、退出权内置的系统（文明、政策、公司）
- 边界衰减：婚姻、教育——角色模糊，点火判定依赖视角，解释力下降
- 类比参考：人生、人际——提议者和应约者在同一主体内，结构相似但机制不同
