# 被动提议对点火趋势的影响

## 主动提议 vs 被动提议

主动提议的点火趋势更稳。周公在权力最稳的时候提礼制，不是被逼的，方案完整——点火和搭房同步，自己真心想执行。主动提议者退路多，选择提方案本身就是判断"这个方案值得提"，方案质量天然更高。

被动提议有三种不同的坑，对应三种点火趋势的偏移：

## 1. 被动提议 + 真心想执行 = 点火趋势不变，但时间窗口窄

戈尔巴乔夫是被经济崩盘逼出来的，但他真心想改。问题是被动意味着外部压力已经很大，留给点火和搭房同步建设的时间窗口极短。时间差被压缩，容易滑向负时间差。戈尔巴乔夫就是被动提议+真心想改+时间窗口太窄，结果先拆旧再建新，直接崩盘。

## 2. 被动提议 + 半截真心 = 点火趋势偏移向"点不足"或"点歪了"

慈禧推洋务是被外敌打出来的，但她推一半就停——推到底会动自己利益。这不是退化，是一开始就没打算真提。结果就是半截提议：只搭了一半房，火只点了一半，认同只覆盖了部分群体。洋务运动本质上是被动提议导致的"点不足"。

更隐蔽的变体：提议者被动提了方案，但把认同锚定在自己能接受的安全位置上，而不是锚定在应约者身上。比如清末"中学为体西学为用"——提议者把认同锚定在"不动根本"上，应约者（需要彻底变革的人）根本不认。这是被动提议导致的"点歪了"。

## 3. 被动提议 + 根本不想提 = 不构成点火

假设提议者被外部压力逼着提了个方案，自己完全不认同，只是做样子——这种"提议"不构成点火，因为提议者不会去搭房，点火和搭房都缺席。应约者看到的是空话，不会应约。

## 核心规则

被动提议本身不决定点火趋势的方向，但会压缩时间窗口、扭曲认同锚点、降低方案完整性。主动提议者有余裕选择"点在哪"，被动提议者的选择空间被压力挤压，容易点在安全位置而不是应约者身上。所以被动提议天然偏向"点不足"或"点歪了"。

这与框架核心判断一致：点火趋势的核心判断是"认同建在哪"，被动提议者倾向于把认同建在自己安全的位置上，而不是应约者身上。
