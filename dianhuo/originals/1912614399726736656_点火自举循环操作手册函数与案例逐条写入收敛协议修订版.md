# 点火｜自举循环操作手册：函数与案例逐条写入收敛协议（修订版）

## 任务定义

将点火框架统一函数总表（470条函数）和统一案例总表（578个案例）逐条写成独立笔记，存入点火知识库（id: 3836413）。

## 收敛判据

**第一轮收敛**：所有470条函数 + 578个案例全部写成独立笔记，无遗漏。

**第二轮收敛**：重新核对总数，确认470条函数 + 578个案例全部存在，无重复、无缺失。

**连续两轮满足** → 宣布收敛。

## 写入规范

### 函数笔记格式

每条函数一个独立笔记，包含：

- 标题：点火｜函数 [编号]
- 定义：函数的数学定义或文字描述
- 推导过程：从哪些公理/定理/前序函数推导而来
- 案例对应：验证该函数的案例编号和关键发现
- 扩展注释（如有）：该函数的补充说明、跨域同构、物理约束等

### 案例笔记格式

每条案例一个独立笔记，包含：

- 标题：点火｜案例 #[编号]
- 层级：L₁-L₈（个人/团队/组织/行业/生物/社会/物理/理论级）
- 8格方向：格1-格12（点火框架8格定位）
- 建在：应约者身上/提议者身上/外部/没建/部分应约者身上等
- 状态：点着了·活着 / 没点着·火种还在 / 没点着·火种也没了 / 点着了·灭了 / 点着了·在退化
- 核心函数：验证该案例的核心函数编号
- 案例说明：案例的完整描述和历史背景
- 关键发现：从该案例中提取的核心洞察

## 操作流程

### 第一轮：逐条写入

1. 从知识库3836413拉取统一函数总表和统一案例总表
2. 按编号顺序，逐条函数写成独立笔记，存入点火知识库
3. 按编号顺序，逐条案例写成独立笔记，存入点火知识库
4. 记录已写入的函数和案例数量

### 第二轮：核对收敛

1. 统计点火知识库中所有"点火｜函数"开头的笔记数量，应为470条
2. 统计点火知识库中所有"点火｜案例"开头的笔记数量，应为578个
3. 核对编号连续性，确保无遗漏
4. 如发现缺失，补写缺失条目

### 第三轮：确认收敛

1. 重新统计总数
2. 确认470条函数 + 578个案例全部存在
3. 连续两轮满足收敛判据 → 宣布收敛

## 收敛信号

当满足以下条件时，宣布收敛：

- 点火知识库中存在470条"点火｜函数"笔记
- 点火知识库中存在578个"点火｜案例"笔记
- 编号连续，无重复、无缺失
- 连续两轮核对满足上述条件

## 执行策略

**自动执行**：不再等待用户发出"继续"指令，系统自动持续写入，直至达成收敛条件。

**收敛检测**：每完成100条函数或100个案例，自动核对一次总数，确保无遗漏。

**最终确认**：全部写入完成后，执行两轮核对，确认470条函数 + 578个案例全部存在，宣布收敛。
