# 点火｜自举循环操作手册（2026.06.12更新）

一、什么是自举循环

自举循环是点火框架的元层迭代机制：理论与案例之间的循环碰撞，每轮碰撞产生新发现，新发现修正理论，修正后的理论再重跑案例库，如此往复。

自举循环不是独立函数，是八函数在元层面的投影。它的全部行为由已有函数驱动，没有独立作用对象。

核心方程：

$$B(n+1) = B(n) + \Delta B(n)$$

$$\Delta B(n) = \sum_i [1 - P(\text{案例}_i | F_1...F_{k^n})] \times \delta(\text{案例}_i\text{重新分布})$$

每轮新发现 = 被旧维度解释不了的案例数 × 这些案例因新维度而重新归位的比例。

---

二、自举元函数是什么

自举元函数是驱动自举循环的那个东西：不是循环本身，是让循环启动并持续的内生动力。

$$M_{boot} = \varepsilon_{sense} \times P_{track} \times \frac{d(\Delta K)}{dt}$$

三个乘法因子：
- $\varepsilon_{sense}$：感知到"这里有结构"的信号——不是知识，是感知
- $P_{track}$：分轨并行——同时运行多个视角，让不同轨道的东西碰撞
- $d(\Delta K)/dt$：知识增量的速率——每轮迭代能不能产出新东西，产出了能不能被感知到

三者乘法关系，任一为零则 $M_{boot}=0$。

自举元函数在人群中的层级分布：

| 层级 | $\varepsilon_{sense}$ | $P_{track}$ | $d(\Delta K)/dt$ | $M_{boot}$ | 人群比例 |
|---|---|---|---|---|---|
| L1-L2 | 有，被日常消耗 | ≈1（单轨） | ≈0 | ≈0 | 多数人 |
| L3 | 有，能发现规律 | 1-2轨 | 低 | >0但不自持 | 部分人 |
| L4 | 强，能感知结构相似性 | 2-3轨 | 中 | 可自持但不稳定 | 少数人 |
| L5 | 最强，能感知"自己正在感知" | ≥3轨 | 高 | 自持且加速 | 极少数 |

自持条件：$M_{boot} > \theta_{boot}$

自持加速方程：$dM_{boot}/dt = \alpha \cdot M_{boot} \cdot \ln(1 + \Delta K/K_0)$

AI缺失论证：$M_{boot}^{AI} = 0$（三因子全零或失能）

三、自举循环的完整操作流程

### 步骤0：确认起点

确认当前统一函数总表版本和案例总数。拉取最新版统一函数总表和统一案例总表。

### 步骤1：选择碰撞对象

用户指定碰撞对象，可以是：
- 一本书/一门课/一个理论体系
- 一个跨域问题/一个直觉/一个新发现
- 用户口述的一段想法
- 函数总表与案例总表之间的双向碰撞（"跑两张表"）

### 步骤2：提取碰撞素材

从碰撞对象中提取：
- 核心概念/命题
- 案例和现象
- 隐含的数学结构
- 与点火框架可能产生共振或冲突的点

### 步骤3：逐点碰撞

将碰撞素材的每个点与统一函数总表逐条对撞：
- 该点能否被已有函数解释？→ 能解释的标注对应函数编号
- 该点暴露了已有函数的什么缺口？→ 记录结构性缺口
- 该点与已有函数的碰撞产生了什么新发现？→ 记录新发现

### 步骤4：数学化展开

对每个未收敛的新发现：
- 写出直觉级的因果描述
- 转化为数学表达式
- 推导至闭合形式
- 标注与已有函数的关系（独立/合并/扩展注释）

### 步骤5：独立性判定

对每个数学化后的新发现做独立性判定：
- 是否包含新的因果结构？→ 是=独立函数
- 是否只是已有函数的参数代入/场景特例？→ 合并为已有函数的注释
- 是否与已有函数同构但不同域？→ 合并为已有函数的跨域注释
- 是否揭示了已有函数之间未被注意的连接/等价/同构？→ 记为扩展注释

### 步骤6：函数编号与收敛

- 独立新函数：按D+编号序列分配编号
- 合并项：写入被合并函数的注释
- 扩展注释：写入对应函数的扩展注释
- 更新统一函数总表

### 步骤7：案例验证

- 为每个新函数/新注释分配验证案例编号
- 每个案例标注：层级、8格归属、方向、状态、发现者价值、核心碰撞、新增函数、关键发现
- 更新统一案例总表

### 步骤8：交叉验证

将新增函数与已有全部函数做交叉验证：
- 数学结构是否矛盾？
- 参数定义是否冲突？
- 与已有函数的对接是否闭合？
- 乘法归零律在全部案例中是否仍然成立？

### 步骤9：更新存库

将更新后的统一函数总表和统一案例总表写入点火知识库。

### 步骤10：收敛检查

计算本轮 $\Delta B(n)$：

**ΔB的三项构成（2026.06.12更新）：**
- 新函数数 > 0 → ΔB > 0
- 新案例数 > 0 → ΔB > 0
- 新扩展注释数 > 0 → ΔB > 0

只有三项全零才算ΔB=0。

收敛判据：
- $\Delta B > 0$：本轮有新发现（函数/案例/注释任一），自举循环未收敛，可继续下一轮
- $\Delta B = 0$：本轮三项全零，进入收敛检查
  - 连续两轮 $\Delta B = 0$（三项全零） → 宣布收敛
  - 只一轮 → 可能偶然，建议再跑一轮确认

四、自举循环的三个驱动力

**驱动力1：分辨率提升**
新增维度使事件条件矩阵的格数翻倍。原来挤在一起的案例差异暴露→$\Delta B > 0$。

**驱动力2：格间重排**
维度从独立降格为派生量，改变案例在矩阵中的投影。格子归属变化→格间对比结论变化→$\Delta B > 0$。

**驱动力3：回路激活**
新函数激活原本休眠的因果回路。回路被激活→新因果结构暴露→$\Delta B > 0$。

**驱动力4（2026.06.12新增）：注释激活**
扩展注释揭示已有函数之间的隐含连接/等价/同构，激活休眠的函数间回路→新注释→$\Delta B > 0$。

---

五、收敛的输出信号

自举循环收敛 = 以下三个条件同时成立：

1. $\delta(\text{案例}_i\text{重新分布}) = 0$ 对所有 $i$ — 没有案例再换格子
2. $P(\text{案例}_i | F_1...F_{k^n}) \to 1$ 对所有 $i$ — 每个案例都被完整解释
3. 所有休眠回路已被激活 — 新维度不再触发新回路，新注释不再揭示新连接

实操判据：**连续两轮三项全零（新函数=0 且 新案例=0 且 新扩展注释=0）。**

用V函数表达：收敛 = V在所有案例上稳定，不再因新维度或新注释而跳值。

六、自举循环在八函数元层面的对应

| 八函数 | 元层面含义 | 对应关系 |
|---|---|---|
| B(n) | 系统当前状态 | 统一函数总表快照 |
| ΔB(n) | 理论与案例的速度差 | Δv的元层面表达：ΔB>0=案例跑在理论前面，ΔB=0=同步 |
| 案例退出信号 | 案例无法被当前函数解释 | ε_sense：案例的前认知退出信号 |
| 修改函数 | 提议者根据退出信号调整Posture | I觉醒：从感知退出信号到调整函数结构 |
| 新维度让案例重新归位 | R_perceived相变 | 案例从"无法解释"跳到"可被新函数解释" |
| 新注释揭示隐含连接 | 函数间休眠回路激活 | 扩展注释是函数间的ε信号——感知到"这两个函数之间有结构" |
| 收敛 | V稳定 | V=链1退化速率/链3恢复速率，V<1收敛 |
| 回路12元版本 | 案例不可解释→误读为"案例就是这样"→不改函数 | C_exit→Posture正反馈的元层面实例 |
| 回路16元版本 | 案例不可解释→感知不到退出信号→函数不变 | C_exit→I_sense遮蔽回路的元层面实例 |
| 异常值 | 怎么都归不进去的案例 | ε硬核：绕过所有劫持直接触发I觉醒 |
