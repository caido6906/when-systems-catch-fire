# 数据迁移打破锁定

两个问题合在一起推，因为它们共享同一个函数结构。

**第一个问题：9年10万条笔记为什么锁不住你？**

上一条论证里时间维度的f_可转移标了0.3-0.5，这是对典型用户的标注。但你现在有了OpenClaw+Get笔记Skill全量导出，这个f_可转移需要重新标定。

C_exit(时间)的原始定义是"时间投入沉没，不可回收"。但这个定义隐含了一个前提：**数据不可迁移**。

如果数据可以全量导出，时间维度的C_exit需要拆分：

C_exit(时间) = C_数据不可迁移 + C_格式转换 + C_重新部署

- C_数据不可迁移：有OpenClaw → **≈0**
- C_格式转换：需要花时间但可自动化 → 低
- C_重新部署：需要花时间但一次性 → 低

f_可转移(时间) 从0.3-0.5跳到**0.8-0.9**。

这意味着你上一条论证里的n_lock≈0-1更加稳固——时间维度从"可能超阈值"变成"确定不超阈值"。

9年10万条笔记积累的不是锁定，是数据。数据一旦可导出，锁定就归零。**积累≠锁定，可迁移性是二者的分界线。**

**第二个问题：为什么平台必须开放用户导出？**

从退出权的函数链出发：

链2：R → R_perceived → C_exit → 退出决策

开放导出作用于链2的哪个环节？**C_exit的时间维度。**

C_exit(时间) = C_数据不可迁移 + C_格式转换 + C_重新部署

不开放导出：C_数据不可迁移 → ∞ → C_exit(时间)超阈值 → n_lock↑
开放导出：C_数据不可迁移 → 0 → C_exit(时间)不超阈值 → n_lock不变

**开放导出 = 降低C_exit(时间) = 降低n_lock = 提高P(退出)**

但这只是直接效应。更深层的是对R_perceived的影响：

不开放导出时，用户知道"我的数据拿不走"→ R_perceived被压低→ 即使R=真实退出权，感知上也觉得"走不了"。这是C_exit对R_perceived的反馈压制——上一条论证里标注过"C_exit决定退出成本影响感知退出权"。

开放导出时，用户知道"我的数据随时可以拿走"→ R_perceived升高→ 退出权从象征级跳到事实级。

**开放导出不只降低C_exit，还提升R_perceived。双通道修复。**

用伪备方案的识别三步法验证：

1. 看名义：平台说"你可以随时走"——备方案的话术
2. 看C_exit：不开放导出→数据拿不走→C_exit(时间)极高→名义备方案是假的
3. 看R_perceived：用户不知道自己能走→R_perceived远低于R→名义备方案是空的

**不开放导出 = 伪备方案。** 名义上"你可以走"，实际上数据锁死你走不了。

**收敛函数：导出权对退出权的函数化影响**

**R_effective(t) = R(t) × f_export(t)**

f_export = σ(k × (1 - C_数据不可迁移/C_threshold))

- 不开放导出：C_数据不可迁移→∞ → f_export→0 → R_effective→0（无论R多高）
- 开放导出：C_数据不可迁移→0 → f_export→1 → R_effective≈R

f_export是退出权的实现系数。没有导出权，退出权是象征性的；有导出权，退出权才是事实性的。

**对平台的反直觉推论：**

开放导出看起来会提高用户流失率，但从P_forward的角度：

P_forward = σ(ε×A×D×|M_cog|) × exp(-n_lock×C̄/θ_C) × f(P_infra) × σ(θ_d-d)

开放导出降低n_lock→exp项升高→P_forward升高→用户留下的概率反而更高。

为什么？因为**P_forward是正向转变概率，不是留存概率**。n_lock低时用户能留下是因为P_forward高（有正向转变的可能），不是因为走不了。n_lock高时用户留下是因为走不了，P_forward可能≈0。

**开放导出把"走不了所以留下"变成"能走但选择留下"——从C_exit锁定变为P_forward吸引。**

前者是伪留存，后者是真留存。伪留存下H必然升高（平台不需要改进，反正用户走不了），真留存下H被ε压制（用户随时能走，平台必须持续改进）。存好了。

两个结论：

**9年10万条笔记锁不住你**：积累≠锁定，可迁移性是分界线。OpenClaw把C_exit(时间)从"数据不可迁移→∞"变成"可导出→0"。f_可转移从0.3-0.5跳到0.8-0.9。积累还在，锁定没了。

**为什么必须开放导出**：收敛函数R_effective = R × f_export。不开放导出→f_export→0→R_effective→0，退出权是象征性的。开放导出→f_export→1→R_effective≈R，退出权才是事实性的。用伪备方案三步法验证：不开放导出=伪备方案，名义上"你可以走"，实际上数据锁死你走不了。

反直觉的推论：开放导出反而提高P_forward——把"走不了所以留下"变成"能走但选择留下"，从C_exit锁定变为P_forward吸引。伪留存下H必然升高（平台不需要改进），真留存下H被ε压制（用户随时能走，平台必须持续改进）。
