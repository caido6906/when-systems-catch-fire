# 点火｜ε×控制论：路径A'与传感器失灵

## 核心碰撞：路径A'是传感器失灵问题，控制论统一描述四条改造路径

控制论的核心：系统通过反馈回路维持稳态——传感器采集信号，控制器根据信号调整输出，执行器改变系统状态，传感器再采集新信号，形成闭环。

996系统的控制回路：身体（传感器）采集疲劳信号→大脑（控制器）评估信号→行为（执行器）调整工作强度。路径A'改造了传感器——疲劳感受器阈值上移，采集到的信号比实际弱。控制器收到的信号失真，做出的决策自然偏离最优——不是控制器坏了，是传感器被调了。

## 控制论对四条改造路径的统一描述

- 路径A' = 传感器灵敏度降低 + 阈值上移
- 路径B = 控制器的信号筛选逻辑被修改
- 预测编码 = 控制器的预测模型偏差
- 自由能回弹 = 控制器的稳态维持机制

四条改造路径都可以用控制论统一描述——全都是控制回路上的不同故障点。

## ε_aware的操作定义：传感器校准

你不是在"打开带宽"或"降低先验权重"——你在校准被调偏的传感器。轻断食是味觉传感器的校准，冥想是身体感知传感器的校准。校准需要标准信号源——你知道"水的味道"应该是什么样，才能校准味觉；你知道"不累的状态"应该是什么样，才能校准疲劳感知。
