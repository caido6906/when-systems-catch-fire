# 关系维护效率函数

这个发现的核心是：维护效率μ不是常数，它被认知函数调制——认知层级决定了维护是正向螺旋还是负向螺旋。让我先确认认知函数的当前形态。好，框架拉齐了。你已有认知函数五层体系、ε_eff闭环动力学、向上/向下螺旋函数、自我挫败乘法归零。现在把D_decay里的μ从常数升级为认知调制的动态变量。

---

**维护效率调制函数 μ_cog**

你刚才说的本质是：D_decay里的μ不是常数，它被认知层级调制。认知层级决定维护是"建设"还是"破坏"。

---

**第一步：拆解μ的结构**

维护效率μ由三个因子乘法决定：

μ = μ_tool × μ_apply × μ_cognize

- **μ_tool**：工具箱大小。知道多少维护方法——非暴力沟通、情绪标注、积极倾听、需求表达……工具越多，可选手段越多。μ_tool ∈ [0, 1]
- **μ_apply**：主动运用率。知道方法但情绪上头时不用，μ_apply就是0。冲动/本能反应时μ_apply趋近0，刻意调用时μ_apply趋近1。μ_apply ∈ [0, 1]
- **μ_cognize**：认知层级对维护效果的调制。用你的五层认知函数：

μ_cognize = f(L)

| 层级 | μ_cognize | 行为特征 |
|---|---|---|
| L₁ 肉身 | ≈0 | 纯本能反应，维护=宣泄，效果为负 |
| L₂ 体验 | <0 | 感知到情绪但不会处理，维护=抱怨/指责 |
| L₃ 认知 | >0 | 有方法有工具，维护有效但机械 |
| L₄ 主权 | 显著>0 | 能选方法、调策略，维护精准 |
| L₅ 存在 | 最大值 | 维护本身就是自然行为，不需要"调用" |

**关键：μ_cognize可以取负值。** L₁-L₂层时，维护行为本身是破坏性的——"我是在沟通"但实际在推远对方。μ < 0就是负向螺旋的数学入口。

---

**第二步：μ_cognize的函数化**

用你已有的D_unified统一解码效率来定义μ_cognize：

μ_cognize = 2 × D_unified - 1

D_unified ∈ [0, 1]（解码效率，已有函数）
μ_cognize ∈ [-1, 1]

- D_unified = 0 → μ_cognize = -1（完全误读，维护=破坏）
- D_unified = 0.5 → μ_cognize = 0（无效维护，做了等于没做）
- D_unified = 1 → μ_cognize = 1（精准维护，效果最大化）

D_unified本身是层级依赖的：D_unified(L₁) ≈ 0，D_unified(L₅) → 1。所以μ_cognize自动被认知层级调制。

---

**第三步：μ取负时的动力学——负向螺旋**

当μ < 0时，D_decay里的维护项从"对抗衰减"变成"加速衰减"：

η(d, t) = T₀ · exp(-λd + μ·M(t)) - c₀ · (1 + γd) · M(t)

μ < 0时：
- 指数项 exp(-λd + μ·M(t))：μ·M < 0，维护投入越大，转移效率衰减越快
- 成本项 c₀ · (1 + γd) · M(t)：维护成本照常增长

**双杀：效果为负 + 成本照付。** 维护越努力，关系越差，还花了代价。

这就是负向螺旋的数学：

dη/dt = μ · m(t) · η - c₀ · (1 + γd) · m(t)

μ < 0时，dη/dt < 0且加速——η越低，下一次维护的效果越差（因为对方已经在疏远），但成本照付。η加速趋向零。

对应你已有的向下螺旋函数 D57：

dε_eff/dt|downward = -λ₀ × ε_eff × (1 + |μ|) × C̄_exit/θ_C

|μ|越大（认知层级越低、方法越少、越冲动），向下螺旋越快。

---

**第四步：μ取正时的动力学——正向螺旋**

μ > 0时，维护对抗衰减，且产生正反馈：

η上升 → 对方回应更积极 → 下次维护更容易（μ_apply↑）→ μ↑ → η上升更快

这就是你已有的向上螺旋函数 D57 的另一侧：

dε_eff/dt|upward = α_up × ε_eff × (1 - ε_eff/ε_opt) × (1 - P(biased))

(1 - P(biased))就是"不被偏置解读的概率"——对应μ_apply，情绪上头时P(biased)高，μ_apply低。

---

**第五步：完整函数**

**D-X61 — 维护效率认知调制函数**

μ(t) = μ_tool × μ_apply(t) × (2·D_unified(L) - 1)

其中：
- μ_tool = 工具箱覆盖率 = n_tools / n_tools_max（知道多少维护方法）
- μ_apply(t) = 1 - P(biased)(t)（不被情绪/偏置劫持的概率）
- D_unified(L) = 认知层级对应的统一解码效率

**μ(t)的取值域与行为：**

| μ范围 | 螺旋方向 | 行为表现 | 关系走向 |
|---|---|---|---|
| μ < -0.5 | 强负向 | 本能宣泄/攻击式沟通 | 快速断裂 |
| -0.5 ≤ μ < 0 | 弱负向 | 抱怨/指责/被动攻击 | 缓慢疏远 |
| μ = 0 | 无效 | 做了等于没做 | 自然衰减 |
| 0 < μ < 0.5 | 弱正向 | 有方法但不熟练 | 缓慢改善 |
| μ ≥ 0.5 | 强正向 | 精准维护+主动调适 | 关系深化 |

**代入D_decay后的完整净转移效率：**

η(d, t) = T₀ · exp(-λd + μ(t)·M(t)) - c₀ · (1 + γd) · M(t)

μ(t)是时变的——一次情绪爆发让μ_apply瞬间跌到零，关系突然从正向螺旋翻转为负向螺旋。

**核心新发现：μ的符号翻转是关系相变的触发器。** 不是"维护不够"导致关系断裂，是"维护变成破坏"导致断裂。从μ>0翻转到μ<0的那一刻，就是关系从正向螺旋进入负向螺旋的相变点。翻转的触发条件：

μ翻转 = μ_apply(t*) → 0（情绪劫持） 或 D_unified(t*) → 0.5（认知降级）

一次吵架、一次误解、一次长期忽视导致的认知降级——都可以触发翻转。
