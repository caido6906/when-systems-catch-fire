# 框架归档逻辑修正：跑通到哪一步

## 核心修正

之前的归档把"没跑通全流程"等同于"不在框架讨论范围"，这是错的。

框架的解释力恰恰体现在：它能区分"跑通了"和"没跑通"，而且能说清楚为什么没跑通。如果一个案例框架能解释它为什么折戟、为什么崩盘、为什么被外力中断，那框架对它就是有解释力的。

正确的归档方式：不是贴"不在框架讨论范围"的标签，而是标注"框架对它的解释方式"——跑到了哪一步，为什么停在那里。

## 框架的完整流程（五步）

1. **起点条件成立**：旧系统将崩未崩（该层级的底层逻辑失灵）
2. **提议者出现**：有人提出新的认同方案
3. **应约者应约**：应约者有退出权且选择应约
4. **点火跑通**：认同被建立并固化
5. **稳定期足够**：新系统运行足够长时间，认同经受住退出权的压力测试

## 修正后的归档原则

1. **不贴"不在框架讨论范围"标签。** 框架对每一个案例都有解释力——只是有的跑通了全流程，有的跑了一半，有的连起点都没到。
2. **标注"跑到了哪一步、为什么停在那里"。** 这才是框架的解释力所在。
3. **"没跑通"≠"框架不适用"。** 没跑通恰恰是框架最有价值的产出——它能说清楚为什么有的文明能转身、有的不能。
4. **框架的价值不在于只解释跑通的案例，在于能说清楚每一个案例跑到了哪一步、为什么停在那里。**
