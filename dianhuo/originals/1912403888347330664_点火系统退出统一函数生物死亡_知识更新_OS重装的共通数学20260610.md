# 点火｜系统退出统一函数：生物死亡·知识更新·OS重装的共通数学（2026.06.10）

来源：三个现象——个体死亡对群体有益、知识封装必须更新、操作系统必然臃肿——是否共享同一个底层数学结构？结论：是。

---

## 一、统一退化函数 V(t)

V(t) = I(t) - M(t)

I(t) = I₀·(1-e^-βt)，信息价值趋于饱和
M(t) = M₀·e^αt，维护成本指数增长

Ω_retire(t) = step(V(t)) = step(I(t) - M(t))

Ω_retire = 0 时单元应退出，Ω_retire = 1 时单元仍有净价值。α/β 比值决定系统寿命——α 越大（维护成本增长越快），t* 越小（寿命越短）。

---

## 二、三域参数映射

### 生物域
- I(t) = 个体对种群的适应性贡献（繁殖、觅食、防御）
- M(t) = 代谢成本 + 损伤累积 + 资源占用
- α 大（损伤累积非线性），β 中（适应能力有上限）
- 退出方式：死亡。对种群价值 = 释放资源 + 释放遗传空间

### 知识域
- I(t) = 封装对当前问题的解释力（能解码多少新信号）
- M(t) = 框架的例外处理成本 + 认知负债（过时结论的纠偏成本）
- α 中（例外累积线性→后期加速），β 低（框架固化后解释力增长极慢）
- 退出方式：新版替代旧版。对知识体系价值 = 清除过时框架，释放认知带宽

### 软件域
- I(t) = OS 对当前硬件和应用生态的适配能力
- M(t) = 补丁堆叠的兼容性成本 + 内存占用 + 启动时间 + 安全漏洞
- α 大（补丁叠补丁，复杂度非线性增长），β 低（架构定型后适配新硬件靠打补丁）
- 退出方式：重装系统 / 换新机。对用户价值 = 清除技术债，恢复运行效率

---

## 三、群体收益函数

个体退出对群体的净收益：

ΔV_group = V_新个体 - V_旧个体 - C_切换

V_新个体 = 新个体/新版本/新系统的净价值
V_旧个体 = 旧个体继续运行的净价值（V<0，负数）
C_切换 = 切换成本（繁殖代价 / 学习新框架 / 迁移数据）

当 ΔV_group > 0 时，退出对群体有利。

---

## 四、修复 vs 替换判据

V_after_repair(t) = I(t) - M(t) + ΔI(R) - R(t)

ΔI(R) = 修复带来的信息价值增量。关键判断：

Ω_replace = step(ΔI(R) - R(t))

Ω_replace = 0 时修复不如替换。三个域的 ΔI(R) 都随时间递减，R(t) 都随时间递增——同一个数学结构。

生物域：DNA修复错误率随年龄增长，ΔI(R) 递减，R(t) 递增→修复不如繁殖新个体
知识域：旧框架修补（加附录、加注释）越来越厚，解释力增量递减→重写不如出新版
软件域：补丁叠补丁，每个补丁引入新兼容问题→打补丁不如重装

---

## 五、底层统一：信息熵增

S_info(t) = S₀ + σ·t + Σ_compatibility(t)

S₀ = 初始信息熵
σ = 基础熵增率（复制错误/例外累积/补丁复杂度）
Σ_compatibility = 兼容性债务（生物：免疫兼容；知识：范式兼容；软件：向后兼容）

Σ_compatibility 是"旧东西不能扔"的代价。生物体不能扔旧基因（功能依赖），知识框架不能扔旧结论（教学依赖），OS 不能扔旧 API（应用依赖）。兼容性债务随时间累积，是 M(t) 指数增长的主要来源。

退出的本质 = 重置 S_info。死亡清零基因熵，新版清零认知熵，重装清零系统熵。不退出 = S_info 只增不减 = 系统必然被自身兼容性债务压垮。

---

## 六、与点火框架的闭合

- Ω_retire = 退出权的系统级实例化。个体退出 = 系统的 ε
- Σ_compatibility = H（遮蔽）在信息论域的实例化。兼容性债务就是遮蔽——旧框架遮蔽了新信号
- ΔI(R) < R(t) = D_immune 失效的判据。修复增量小于修复成本时 D_immune = 0，退化不可逆
- V(t) 穿过零点 = 认知域的 θ₃ 跳变。从"资产"跳到"负债"的临界点
- 信息熵 S_info(t) 只增不减 = 八函数中 H(t) 只增不减的物理基础。H 的增长不是认知域的特有现象，是所有信息处理系统的热力学必然

---

## 七、推论

1. 任何信息处理系统都有寿命，寿命由 α/β 比值决定。降低 α（减慢维护成本增长）或提升 β（加速信息价值增长）可以延长寿命，但不能消除寿命上限
2. "向后兼容"是所有系统的慢性毒药。兼容性债务 Σ_compatibility 是 M(t) 指数增长的主因——生物的基因冗余、知识的范式惯性、软件的 API 遗留，同一个东西
3. 重置 S_info 是唯一能让系统重新获得净价值的操作。死亡、出新版、重装——形式不同，数学相同
4. 不允许退出的系统（海拉细胞、永不更新的教材、永不重装的OS）等价于 S_info 单调递增的系统，终局是退化稳态——还在运行，但净价值为零
