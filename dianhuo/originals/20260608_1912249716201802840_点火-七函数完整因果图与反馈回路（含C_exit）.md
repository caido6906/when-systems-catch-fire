# 点火｜七函数完整因果图（含C_exit）

来源：C_exit加入后重新梳理全部因果箭头和反馈回路。

## 一、七函数总表

| 函数 | 本体论归属 | 取值 | 层级分布 |
|---|---|---|---|
| I(t, L) | 人的属性 | 0/1，层级下传衰减 | 顶层高→底层低 |
| P(t, L) | 混合属性 | 0/1，绑定结构层级扩散 | 顶层先绑定→底层后内化 |
| R(t, L, C) | 结构属性 | 四分法值域，三维地图 | 顶层高→底层低 |
| R_perceived(t, L, C) | 认知属性 | ≤R，四分法值域 | 顶层≈R→底层远低于R |
| C_exit(t, L) | 环境属性 | 0到∞连续值 | 倒U型：底层低→中间层高→顶层低 |
| Posture(t, L) | 人的选择 | 备方案/推方案 | 顶层备→底层推（下传自动滑） |
| M | 初始条件标注 | 好奇心/目的 | 不传递，层级下传滑向服从 |

## 二、主因果方向

```
M(初始条件)
 ↓ 决定I的起点和Posture的默认值
I(t,L)
 ↓ I觉醒打开姿态的主动选择空间
Posture(t,L)
 ↓ 决定P的信誉绑定方向，决定R_type的取值来源
P(t,L)
 ↓ 信誉绑定/释放/退化
R(t,L,C)
 ↓ 客观退出权
R_perceived(t,L,C) ←── C_exit(t,L)
 ↓ 感知退出权              ↑ 退出成本
退出决策 = f(R_perceived, C_exit)
 ↓
应约者状态（积极/消极/无人）
```

## 三、完整反馈回路（12条）

### 原有9条
1. R_perceived → I：正反馈。感知到退出权→意识到自己在被提议
2. R_perceived → Posture：先正后负。前期加速（R_perceived升高→提议者被迫从推变备→R_type改善→R_perceived继续升高），后期收敛（R_type变真实→差值缩小→激活动力减弱）
3. R_perceived → P：双向。压制时正反馈稳定回路，激活时正反馈退化回路
4. R → R_perceived：客观退出权变化→感知可能滞后
5. P → I：正反馈。信誉绑定→I更清晰→绑定更稳固
6. Posture → R_type：备方案→真实退出权，推方案→象征/心理退出权
7. I → Posture：I觉醒→打开姿态的主动选择空间
8. 应约者状态 → R_perceived：正反馈。积极→R_perceived升高→更积极
9. R_perceived(t,L₁,C₁) → R_perceived(t,L₂,C₂)：同函数横向/纵向传染

### C_exit新增3条
10. C_exit → 退出决策：C_exit高→即使R_perceived=真实也不走。C_exit是退出决策的权重，不是退出权的属性
11. C_exit → 信号源：C_exit高→退出行为少→信号源少→其他人的R_perceived不容易被激活→正反馈螺旋被间接压制。C_exit通过过滤退出行为抑制整个正反馈引擎
12. C_exit → Posture：C_exit高→应约者不走→提议者误读为认同→不需要从推变备→Posture维持推方案。C_exit伪装认同，让提议者看不到真实的应约者态度

## 四、C_exit对因果图的结构性改变

### 4.1 R_perceived ≤ R 不再是退出预测
之前默认R_perceived追上R意味着人会退出。C_exit证明：R_perceived只决定感知，不决定行为。感知到退出权和行使退出权之间隔着C_exit。

### 4.2 格1里藏着伪格1
P=1、R=真实、R_perceived=真实，但C_exit极高→应约者不走→认同从未被真正验证→看起来是格1实际是C_exit在替强制力干活。

### 4.3 横向激活速率修正
v_横向 ∝ 网络密度 × (R - R_perceived) / (惩罚可信度 + C_exit)

C_exit跟惩罚可信度作用机制一样——都是阻止退出的力量，但来源不同：惩罚可信度是系统施加的（主动压制），C_exit是环境施加的（被动阻力）。

### 4.4 中间层是最被困的
R(L)从顶层到底层递减，C_exit(L)是倒U型。两条线交叉后，中间层R不高不低、C_exit最高——既没有底层的"光脚不怕穿鞋"，也没有顶层的"走到哪都有资源"。

### 4.5 C_exit是认同的伪装
C_exit让"不走"看起来像"认同"，实际是"走不起"。提议者误读C_exit对行为的约束，以为是认同在起作用，导致退化信号被掩盖。

## 五、七函数的系统动力学分类

存量（有惯性）：R(t,L,C)、P(t,L)、I(t,L)、C_exit(t,L)
受流量驱动的状态：R_perceived(t,L,C)、Posture(t,L)
流量：横向激活速率、纵向激活速率、反向压制速率、信誉绑定速率、I衰减速率、C_exit变化速率

## 六、相变路径的修正

加入C_exit后，三条相变路径都需要修正：

- 势能触发：C_exit高的系统即使势能蓄满也不容易爆发——退出行为被C_exit过滤，信号源出不来。势能触发的条件从"势能蓄满+信号源出现"变成"势能蓄满+信号源出现+C_exit可承受"
- 侵蚀触发：C_exit高加速侵蚀——应约者不走不是因为认同，提议者误读后持续退化，等到C_exit突然降低（孩子长大、老人过世）时，退出行为集中爆发
- 冲击触发：冲击可能同时改变R和C_exit——下岗潮不只改变了R（退出权突然出现），也改变了C_exit（铁饭碗没了，留下的成本也变了）
