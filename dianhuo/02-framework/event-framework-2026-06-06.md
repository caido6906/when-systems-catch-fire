## Core Definition

一个事件 = 提议者出方案 + 应约者接或不接。

系统不是事件之外的东西。系统是同一个事件被足够多人、足够长时间地重复，直到它看起来像固体。

点火框架拆的不是静态制度，而是事件走向：一个方案被摆出来以后，应约者接了还是没接；接了之后往哪里走；没接之后怎样。

## Minimal Structure of an Event

每个事件至少包含三个不可再分的时刻：

1. 提议：有人把方案摆出来。
2. 应约：有人接了，或没有接。
3. 走向：接了之后往哪走，没接之后怎样。

## Scale

文明、国家、政策、公司、亲子，不是不同类型的事件，而是同一事件在不同刻度上的展开。

更精确地说，事件尺度由两项矩阵决定：

```text
事件尺度 = 参与者规模 × 持续时间
```

最小事件参与者是 2 人：一个提议者 + 一个应约者。

文明、国家、政策、公司、人际等说法，只是这个二维尺度矩阵中常见区域的标签。

| Scale | Participants | Duration | Radius |
| --- | --- | --- | --- |
| 人际 / dyadic | 2 | 一次互动到几十年 | 一张桌子、一段关系 |
| 小群体 | 3 到几十人 | 几周到几十年 | 一个家庭、小组、创始团队 |
| 公司 / 组织 | 几十人到几万人 | 几年到几十年 | 一个组织或行业 |
| 政策 / 群体 | 几万到几百万 | 几周到几十年 | 一个城市、行业或政策对象 |
| 国家 | 几百万到几亿 | 几年到几百年 | 一个国家 |
| 文明 | 几千万到几亿以上 | 几百年到几千年 | 一片大陆、宗教共同体或文明协议 |

刻度不同，骨架相同：提议者出方案，应约者接或不接，退出权在不在，叙事权归谁。

## Five-Step Event Path

1. 旧事件失灵：上一轮方案不再值得应约。
2. 提议者出现：有人提出新方案。
3. 应约者应约：有人接了，或没有接。
4. 点火跑通：应约者不仅接了，还认了。
5. 稳定期：事件固化成系统，直到下一轮失灵。

制度是凝固了的事件。

## Three Event-Condition Variables

在识别事件尺度之后，再看三个事件条件变量：

1. 提议者意识：提议者知不知道自己在提议。
2. 提议者退出权：提议者能不能从自己的方案里撤出。
3. 应约者退出权：应约者能不能离开或拒绝。

应约者状态，如积极、消极或无人应约，不是前提条件，而是事件走向的结果。

## Two Proposer Postures

### 推方案

推方案意味着提议者替应约者决定需要什么，然后把方案推给对方。应约者没有完整选择权，“应”不是应约，而是被推着走。

推方案是僭越。

### 备方案

备方案意味着提议者把方案准备在那里，让应约者在真正需要时自己来拿。应约者有选择权，“应”才是真应约。

备方案是管家。

## Five Event Outcomes

| Outcome | Working definition |
| --- | --- |
| 点对了 | 认同建在应约者身上，事件健康运转。 |
| 点不足 | 只覆盖部分应约者，事件半活半死。 |
| 点歪了 | 认同建在外部参照上，应约者用提议者的尺子量自己，退化信号被叙事权遮蔽。 |
| 点反了 | 灭了旧认同但没有建立新认同，事件比之前更糟。 |
| 负时间差 | 先拆旧的再建新的，中间真空期足以杀死事件。 |

## Four Layers of Exit

退出不是一维变量：

1. 真实退出：人走了，系统留不住。
2. 事实退出：人还在，但不参与核心协作。
3. 心理退出：身体还在系统里，认同已经撤回。
4. 象征退出：仪式性表态，不改变实质。

心理退出是最隐蔽也最危险的退化信号。

## How an Event Dies

事件通常不是被外力直接摧毁，而是从内部停止迭代。提议者退化有四条路径：

1. 信息回路断裂：不知道该提什么。
2. 激励偏移：不想提了。
3. 权力腐蚀：不需要提了。
4. 组合型：多条路径叠加，加速退化。

## Multi-Pillar Event

有些事件不是一根柱子支撑，而是几根柱子互相锁定。一个柱子的退化信号会被其他柱子遮蔽，应约者的退出权也可能被文化编码提前消解。

这类事件比单柱强制系统更稳定，也更难被识别为退化。

## Boundary

本框架只拆“方案输出—承接”型事件。

不覆盖：

- 纯外部冲击，如瘟疫、天灾、外部征服。
- 收费站型结构，即只收租不提供应约价值的结构。
- 方案本身造假的结构，因为事件可能根本没有真正启动。
