# Concept map

## Core chain

旧事件失灵 -> 提议者提出方案 -> 应约者接或不接 -> 关键人物 / Key Person 做出关键动作 / Key Action -> 退出权和叙事权结构改变 -> 燃烧条件 / Combustion Conditions 改变 -> 点对了、点不足、点歪了、点反了或负时间差

## Core concepts

| 中文术语 | English term | Role in the framework | Related concepts |
| --- | --- | --- | --- |
| 点火框架 | Ignition Framework | 解释新系统如何被应约者主动认领的结构 | 系统重写、燃烧条件 |
| 事件框架 | Event Framework | 将系统理解为重复并凝固的“提议—应约—走向”事件 | 提议者、应约者、五步走向 |
| 事件 | Event | 提议者出方案，应约者接或不接，随后产生走向 | 系统、制度、事件刻度 |
| 事件尺度矩阵 | Event Scale Matrix | 用参与者规模 × 持续时间判断事件落在哪个尺度区域 | 参与者规模、持续时间、事件层级 |
| 参与者规模 | Participant Scale | 真正被卷入“提议—应约—走向”的应约者范围 | 最小事件、事件尺度矩阵 |
| 持续时间 | Duration Scale | 事件走向跑完、凝固、退化或被替代所需的时间 | 稳定期、事件尺度矩阵 |
| 事件层级 | Event Layer | 人际、公司、政策、国家、文明等尺度矩阵中的常见区域标签 | 事件尺度矩阵 |
| 关键人物 | Key Person | 在关键时期改变系统燃烧条件的人或群体 | 提议者、关键动作 |
| 关键时期 | Key Period | 旧系统失灵、新系统尚未稳固的窗口 | 稳定期、延迟点火 |
| 关键动作 | Key Action | 改变应约者预期和系统燃烧条件的行动 | 认同系统、退出权 |
| 燃烧条件 | Combustion Conditions | 新系统被接住所需的结构性条件 | 退出权、认同系统 |
| 协作系统 | Collaboration System | 制度、流程、组织、资源配置、技术结构 | 搭房 |
| 认同系统 | Recognition System | 身份、意义、参与感、归属感、共同方向 | 点火 |
| 提议者 | Proposer | 提出新系统、新秩序、新认同方案的一方 | 关键人物 |
| 应约者 | Responder | 必须决定是否进入或维持新系统的一方 | 退出权 |
| 提议者意识 | Proposer Awareness | 提议者是否知道自己正在提出方案 | 8 格事件条件 |
| 提议者退出权 | Proposer Exit Right | 提议者能否从自身方案中撤出 | 8 格事件条件、承诺信号 |
| 事件条件矩阵 | Event Condition Matrix | 提议者意识 × 提议者退出权 × 应约者退出权 | 8 格、事件走向 |
| 提议者姿态 | Proposer Posture | 推方案或备方案，是事件条件矩阵内的次级变量 | 退出权真假、格间滑动 |
| 格间滑动 | Cell Drift | 同一事件内部从一个条件格滑向另一个条件格 | 提议者退化、退出权变化 |
| 退出权 | Exit Right | 应约者承诺被测试的压力条件 | 真实退出权、心理退出权 |
| 推方案 | Pushed Proposal | 提议者替应约者决定需要什么并推给对方 | 僭越、服从、心理退出 |
| 备方案 | Prepared Proposal | 提议者准备方案，让应约者在需要时自主取用 | 管家、真应约、培育 |
| 点对了 | Correct Ignition | 认同建在应约者身上，事件健康运转 | 应约者认同 |
| 点不足 | Insufficient Ignition | 只覆盖部分应约者，事件半活半死 | 局部认同 |
| 点歪了 | Misaligned Ignition | 认同建在外部参照上，退化信号被叙事权遮蔽 | 外部锚定、叙事权 |
| 点反了 | Reverse Ignition | 灭了旧认同但没建新认同，事件变得更糟 | 负时间差 |
| 负时间差 | Negative Time Gap | 旧系统先被拆，新系统未跑通，中间真空杀死事件 | 改革风险 |
| 多柱事件 | Multi-Pillar Event | 多个权力、文化或资源柱互相遮蔽退化信号 | 无退出权、文化编码 |
| 延迟点火 | Delayed Ignition | 点火未立即发生，但条件可能继续积累 | 关键时期、稳定期 |
| 失败点火 | Failed Ignition | 协作系统建立但认同系统未建立 | 伪点火、反例 |
| 伪点火 | False Ignition | 表面热烈但应约者未真正认领 | 象征退出权、心理退出权 |

## Relationship notes

- 协作系统解决“能不能一起做事”。
- 认同系统解决“愿不愿意把自己放进去”。
- 事件框架解决“方案被提出以后，应约者接或不接，随后走向哪里”。
- 事件尺度矩阵先解决“这个事件牵涉多少参与者，持续多久”。
- 人际、公司、政策、国家、文明是尺度矩阵中的常见区域标签，不是第一层分类。
- 退出权永远看应约者，不看提议者。
- 提议者退出权不等于应约者退出权，但可以成为提议者承诺是否真实的信号。
- 关键动作只有在改变燃烧条件时，才是点火框架意义上的关键动作。
- 推方案容易把应约伪装成服从；备方案更接近真应约。
- 提议者姿态不是第七维度，而是退出权真假和格间滑动的解释变量。
- 失败点火和伪点火都可能有高度可见的制度或仪式，但缺少应约者主动认领。
- 点歪了不容易被识别，因为叙事权可能由提议者垄断。
