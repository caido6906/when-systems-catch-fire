# Concept map

## Core chain

旧系统失灵 -> 提议者出现 -> 关键人物 / Key Person 做出关键动作 / Key Action -> 应约者在退出权压力下选择是否进入 -> 燃烧条件 / Combustion Conditions 改变 -> 点火、延迟点火、失败点火或伪点火

## Core concepts

| 中文术语 | English term | Role in the framework | Related concepts |
| --- | --- | --- | --- |
| 点火框架 | Ignition Framework | 解释新系统如何被应约者主动认领的结构 | 系统重写、燃烧条件 |
| 关键人物 | Key Person | 在关键时期改变系统燃烧条件的人或群体 | 提议者、关键动作 |
| 关键时期 | Key Period | 旧系统失灵、新系统尚未稳固的窗口 | 稳定期、延迟点火 |
| 关键动作 | Key Action | 改变应约者预期和系统燃烧条件的行动 | 认同系统、退出权 |
| 燃烧条件 | Combustion Conditions | 新系统被接住所需的结构性条件 | 退出权、认同系统 |
| 协作系统 | Collaboration System | 制度、流程、组织、资源配置、技术结构 | 搭房 |
| 认同系统 | Recognition System | 身份、意义、参与感、归属感、共同方向 | 点火 |
| 提议者 | Proposer | 提出新系统、新秩序、新认同方案的一方 | 关键人物 |
| 应约者 | Responder | 必须决定是否进入或维持新系统的一方 | 退出权 |
| 退出权 | Exit Right | 应约者承诺被测试的压力条件 | 真实退出权、心理退出权 |
| 延迟点火 | Delayed Ignition | 点火未立即发生，但条件可能继续积累 | 关键时期、稳定期 |
| 失败点火 | Failed Ignition | 协作系统建立但认同系统未建立 | 伪点火、反例 |
| 伪点火 | False Ignition | 表面热烈但应约者未真正认领 | 象征退出权、心理退出权 |

## Relationship notes

- 协作系统解决“能不能一起做事”。
- 认同系统解决“愿不愿意把自己放进去”。
- 退出权永远看应约者，不看提议者。
- 关键动作只有在改变燃烧条件时，才是点火框架意义上的关键动作。
- 失败点火和伪点火都可能有高度可见的制度或仪式，但缺少应约者主动认领。
