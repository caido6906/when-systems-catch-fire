# 点火框架核心案例集

> 版本：2026-06-07 / 核心案例集 v0.1（框架）
> 目标：12 个理论验证案

---

## 案例选择原则

1. **小而硬**：材料可靠 > 数量庞大
2. **覆盖全矩阵**：点对、点不足、点歪/点反、边界案例各 3 个
3. **跨领域跨规模**：不同领域、不同尺度，检验框架的迁移能力
4. **可复核**：每个案例的资料来源可被第三方查验

---

## 案例模板

每个案例按以下统一模板撰写：

```
## [案例编号] [案例名称]

### 1. 旧系统问题
[旧系统是什么？出了什么问题？燃烧条件是否成立？]

### 2. 提议者与新系统
[谁提出了什么？提议者意识强弱？提议者有无退出权？]

### 3. 应约者
[应约者是谁？有无真实退出权？关键人物是谁？]

### 4. 认领/拒绝过程
[应约者做了什么关键动作？为什么接住/没接住？]

### 5. 竞争解释对照
[博弈论怎么解释？制度理论怎么解释？创新扩散怎么解释？社会认同怎么解释？]

### 6. 点火框架的增量解释
[点火框架比上述理论多解释了哪一步？]

### 7. 编码结果
| 字段 | 值 |
|------|-----|
| 提议者意识 | 强 / 弱 |
| 提议者退出权 | 强 / 弱 |
| 应约者退出权 | 强 / 弱 |
| 燃烧条件 | 满足 / 不满足 / 部分满足 |
| 最终应约状态 | 积极应约 / 消极应约 / 无人应约 / 反向应约 |
| 点火走向 | 点对 / 点不足 / 点歪 / 点反 / 负时间差 |
| 后续稳定性 | 短期成立 / 中期成立 / 长期成立 / 退出权释放后坍塌 |

### 8. 证据质量
[资料来源、可靠性、潜在偏差]

### 9. 开放问题
[本案例中尚未解决的分析问题]
```
