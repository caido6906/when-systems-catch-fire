# 点火框架适用边界：三张表

> 版本：2026-06-07 / 适用边界 v1.0

---

## 判据：什么情况可以叫「点火」？

以下条件**必须同时满足**，缺一则不适用点火框架：

1. **存在旧系统**：旧系统是一个可命名的协作结构或秩序，且其承载力正在下降
2. **存在新系统**：有人（提议者）提出了可被认领的新协作系统
3. **存在应约者**：有可识别的应约者群体，面对「是否把自己放进新系统」的选择
4. **退出权压力**：应约者可以在不接受新系统的情况下继续存在（退出权真实或至少可识别）
5. **主动认领**：应约者做出了可观察的、不可简单还原为强制或纯利益交换的认领行为

## 表一：可解释清单

| 类 | 特征 | 典型案例方向 | 解释重点 |
|----|------|------------|---------|
| 新秩序认领 | 旧系统崩塌，新系统悬空，应约者在可退出状态下主动走进去 | 改革开放初期农民包产到户、东欧转型中某国市场改革被民间认领 | 退出权压力下的主动选择 |
| 关键人物突破 | 关键应约者的认领行为改变了其他潜在应约者的判断 | 早期采纳者带动整个群体 | 关键人物的信号放大效应 |
| 点不足场景 | 协作系统建成但认同系统未形成，系统运行依赖外部推力 | 补贴驱动的产业园区、行政命令推动的合作社 | 表面应约与真点火的区分 |
| 点歪场景 | 应约者认领了被曲解或改造过的系统版本 | 改良被激进派劫持、好设计被错误执行 | 提议者意识与应约者改造的冲突 |
| 点反场景 | 新系统的提出反而强化了对旧系统的认领 | 外来制度威胁激发本土制度捍卫 | 负反馈机制 |
| 负时间差 | 旧系统已死，新系统未生，协作真空 | 政权崩塌后的治理真空期 | 时间差的破坏性 |
| 表面应约 | 有认领行为但退出权测试后暴露为假认领 | 强制参与的组织在管制放松后迅速崩解 | 认同真实性的检验 |
| 无意识点火 | 提议者无意识但客观上产生了被认领的新系统 | 某些民间自组织、开源社区 | 区分事后追认与当时意图 |
| 制度转换时刻 | 旧制度合法性衰减，新制度在竞争中赢得认领 | 货币改革、法律体系切换 | 合法性悬空期的选择 |
| 跨尺度点火 | 同一事件在不同尺度上有不同点火结果 | 产品→制度：iPhone改变了电信业，但没改变国家制度 | 尺度锚定不可混用 |

## 表二：不可解释清单

| 类 | 特征 | 为什么不是点火 | 该用什么解释 |
|----|------|--------------|------------|
| 纯技术效率提升 | 换了更好的工具，协作结构没变 | 旧系统没有被新系统替代，只是旧系统的升级 | 技术采纳模型 |
| 市场销售增长 | 卖得多了 | 消费者可选择购买，不是「认领一个新秩序」 | 市场营销学 |
| 强制执行成功 | 不许选，必须做 | 应约者没有退出权，认领无法判断真假 | 权力/强制理论 |
| 短期流量扩散 | 爆了又冷了 | 没有协作系统建成，也没有认同系统 | 传播学/网络效应 |
| 个人情绪感染 | 一群人突然一起哭/一起嗨 | 没有新系统被提出，没有协作结构变化 | 社会心理学 |
| 普通需求满足 | 饿了吃饭，冷了穿衣 | 没有旧系统不足→新系统认领的链条 | 经济学 |
| 事后成功叙事 | 赢了才说是「点火」，输了就当没提过 | 因果倒置，用结果定义原因 | 历史叙事批判 |
| 组织常规动员 | 领导喊口号，下属照做 | 没有应约者的主动选择，是组织流程 | 组织行为学 |
| 文化流行现象 | 一首歌火了，一个梗传开了 | 没有协作系统，也没有认同系统 | 文化研究/模因论 |
| 自然演化 | 慢慢变的，没有可识别的提议者和认领时刻 | 没有事件性，无法识别关键动作 | 演化理论 |
| 纯利益交换 | 你给我钱，我给你货 | 没有「新秩序」的认领，是交易 | 经济学 |
| 模仿/抄袭 | 看到别人成功跟着做 | 旧系统（先行者）不存在不足，追随者没有在退出权压力下做选择 | 创新扩散 |

## 表三：边界案例清单

这些案例可以分析但必须谨慎——每个都必须标注尺度，不得跨尺度偷换结论。

| 案例 | 尺度 | 可分析的点火元素 | 主要风险 | 当前判断 |
|------|------|----------------|---------|---------|
| 乔布斯/iPhone | 产品层 | 旧手机范式不足→新范式提出→用户选择放弃实体键盘 | 退出权过于充分（随时可买安卓），可能只是普通产品采纳 | 可分析，但不要叫「文明级点火」 |
| 福特/Model T | 产品层 | 旧出行方式不足→流水线汽车→中产阶级认领汽车生活 | 同上。福特自己说的也是「造更好的车」，不是「搭新秩序」 | 可分析，提议者意识存疑 |
| 改革开放 | 制度层/国家层 | 旧经济体制危机→市场机制提出→农民/企业家主动认领 | 退出权复杂：不参与农村改革意味着继续挨饿，是「选择」还是「别无选择」？ | 制度层核心案例，但退出权需要精细分期 |
| 苏联解体后东欧转型 | 制度层/国家层 | 旧系统崩塌→多种新系统悬空→各国不同结果 | 退出权被地缘政治约束严重扭曲 | 天然比较案例组，但需区分「选新系统」和「被迫选」 |
| 个人 Agent 写作系统 | 个人生产系统层 | 旧写作方式（手写/纯脑力）不足→AI协作新系统提出→用户主动认领 | 提议者和应约者是同一个人，退出权概念需要重新定义 | **本项目的元案例**——自己就是点火框架的测试对象 |
| 公众号写作 | 个人产品层 | 旧表达渠道不足→公众号平台→写作者认领新渠道 | 更接近创新扩散，协作系统和认同系统都不显著 | 更适合用创新扩散解释 |
| 华夏文明转身 | 文明层/身份层 | 文明级危机→新身份叙事→共同体重新定义「我们是谁」 | 全部判断为假说。退出权在文明层几乎不可定义 | **不可写成定论，必须降级为待检验假说** |
| 开源社区 | 组织层 | 旧软件开发模式不足→开源协作系统→开发者认领 | 退出权极其充分，点对案例极佳 | 低风险，可做核心案例 |
| 宗教改革 | 文明层/组织层 | 旧教会体系不足→新教系统提出→信徒认领 | 退出权被灵魂惩罚威胁扭曲 | 文明层典型边界案例 |
| 维基百科 | 组织层 | 旧百科全书模式不足→wiki协作系统→编辑者认领 | 无意识点火还是集体行动？提议者意识存疑 | 可与 Olson 集体行动理论对照的好案例 |

## 使用规则

1. 任何声称「这是点火」的判断，必须先在表三中定位其尺度
2. 任何落在表二的现象，不得使用点火框架解释
3. 表三中的案例，每次讨论时必须标注尺度，禁止跨尺度类比
4. 新案例先入表三（边界案例），通过编码和反例测试后，才可升入表一
