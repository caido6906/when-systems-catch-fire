## 每个案例的编码字段

每个案例必须填写以下字段才能进入矩阵：

### 基本信息
| 字段 | 说明 | 示例 |
|------|------|------|
| `case_id` | 唯一标识 | `CN-001` |
| `case_name` | 案例名称 | 安徽小岗村 |
| `time_period` | 时间范围 | 1978 |
| `domain` | 领域 | 农业 / 政治 / 科技 / 商业 / 社会 |
| `scale` | 规模 | 村 / 县 / 省 / 国 / 跨国 |

### 提议者维度
| 字段 | 取值 | 说明 |
|------|------|------|
| `proposer_identity` | 文本 | 谁提出了新系统 |
| `proposer_awareness` | 强 / 弱 | 提议者是否意识到自己在搭一个新秩序 |
| `proposer_exit_power` | 强 / 弱 | 提议者能否选择离开 |
| `proposer_awareness_evidence` | 文本 | 可观察证据 |
| `proposer_exit_evidence` | 文本 | 可观察证据 |

### 旧系统维度
| 字段 | 取值 | 说明 |
|------|------|------|
| `old_system_name` | 文本 | 旧系统是什么 |
| `old_system_decline_type` | 渐进 / 突然 / 慢性 / 未衰退 | 旧系统怎么不行了 |
| `burning_condition` | 满足 / 不满足 / 部分满足 | 燃烧条件是否成立 |
| `burning_evidence` | 文本 | 可观察证据 |

### 应约者维度
| 字段 | 取值 | 说明 |
|------|------|------|
| `respondent_identity` | 文本 | 应约者是谁 |
| `respondent_exit_power` | 强 / 弱 | 应约者有没有真实退出权 |
| `respondent_exit_evidence` | 文本 | 可观察证据 |
| `key_figure_name` | 文本（或「无」） | 关键人物 |
| `key_figure_action` | 文本 | 关键动作描述 |

### 结果维度
| 字段 | 取值 | 说明 |
|------|------|------|
| `final_response` | 积极应约 / 消极应约 / 无人应约 / 反向应约 | 最终应约状态 |
| `ignition_outcome` | 点对 / 点不足 / 点歪 / 点反 / 负时间差 / 表面应约 / 无火 | 点火走向 |
| `stability` | 短期成立 / 中期成立 / 长期成立 / 退出权释放后坍塌 | 后续稳定性 |
| `evidence_quality` | 高 / 中 / 低 | 证据可靠性评级 |
| `coder_confidence` | 高 / 中 / 低 | 编码者自评信心 |

## 编码规则

### 进入矩阵的条件
1. 旧系统与新系统都能被命名
2. 提议者行为可被独立验证
3. 应约者的退出权状态可被评判
4. 认领或不认领的行为有直接或间接证据

### 排除条件
1. 无法确定旧系统是否真的「旧」（不能假定任何系统都会燃烧）
2. 无法确定新系统是否真的「新」（不能假定任何变化都是系统更迭）
3. 应约者不存在或不可识别（集体无意识行为不进入矩阵）

### 争议处理
- 两位以上独立编码者打分不一致时，标记为 `争议案例`，保留分歧编码
- 单一案例可以跨格（如「开始于格3，后来进入格1」），需标注时间序列
