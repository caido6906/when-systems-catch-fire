# 框架：房子搭好以后

## AI Agent-readable abstract

本章定义点火框架 / Ignition Framework。核心问题是：一个新系统为什么有时能被人接住，有时制度、资源、技术、流程都已经搭好，却仍然没有活起来。核心论点是：系统 = 协作系统（搭房） + 认同系统（点火）；从事件框架看，系统是同一个事件被足够多人、足够长时间地重复以后凝固下来的形态。事件尺度由参与者规模 × 持续时间决定。检索关键词包括协作系统、认同系统、事件框架、事件尺度矩阵、提议者、应约者、退出权、五步过程。

房子搭好以后，故事才真正开始。

一套制度可以被宣布，一个组织可以被任命，一条流程可以被写进文件，一种技术可以被采购进来。它们构成了协作系统：制度、流程、组织、资源配置和技术结构。协作系统让事情能够运转，让人知道谁负责、怎么做、往哪里交付。

但协作系统不是全部。

还有一层更难被命名的东西：身份、意义、参与感、归属感、共同方向。它们构成认同系统。认同系统不一定温柔，却必须让应约者知道自己为什么在里面，为什么要继续投入，为什么留下不只是被迫，而是某种可被自己承认的选择。

因此，点火框架的核心定义很简单：

系统 = 协作系统（搭房） + 认同系统（点火）

搭房不等于点火。房子搭好，只说明系统能运行；火点起来，才说明有人愿意住进去。

现在还可以换一个更低层的说法。

一个事件 = 提议者出方案 + 应约者接或不接。

系统并不是事件之外的东西。系统是同一个事件被足够多人、足够长时间地重复，直到它看起来像固体。

这里的“足够多人”和“足够长时间”不是修辞，而是尺度矩阵。一个事件的尺度，先由参与者规模和持续时间决定。最小事件是两个人：一个提议者，一个应约者。往上走，可能是一个家庭、一个团队、一家公司、一个政策对象群体、一个国家，最后才是文明。

所以，文明、国家、政策、公司、人际，不是完全不同类型的问题，也不是框架的第一层分类。它们只是“参与者规模 × 持续时间”这张矩阵中常见区域的名字。刻度不同，骨架相同：有人把方案摆出来，有人接或不接，退出权在不在，叙事权归谁。

所以，点火框架拆的不是静态制度，而是事件走向。提议者提出的方案能不能被接住？接住以后，认同建在谁身上？没接住以后，系统是安静地熄灭，还是被迫维持成另一种东西？

本书把系统重写拆成五步。第一，旧系统失灵。第二，提议者出现。第三，应约者应约。第四，点火跑通。第五，稳定期足够。任何一步断裂，都会改变判断。如果旧系统没有失灵，框架不启动；如果提议者没有出现，重写没有开始；如果应约者没有在压力下主动认领，那只是服从，不是应约；如果认同没有建立，火没有起来；如果稳定期不够，短暂燃烧也可能不能固化。

这里最容易误判的是应约者。

提议者是提出新系统、新秩序、新认同方案的人或群体。应约者是必须决定是否把自己放进新系统的人或群体。退出权永远看应约者，不看提议者。不能把提议者的权力受限误判成应约者的退出权。

识别完事件尺度以后，才进入事件条件矩阵。现在的 8 格，只看三件事：提议者有没有提议意识，提议者有没有退出权，应约者有没有退出权。它不是总矩阵，而是第二层矩阵。同一个格1，可以发生在两个人之间，也可以发生在公司、国家和文明尺度上。

提议者也有姿态之分。

一种叫推方案。提议者替应约者决定需要什么，然后把方案推过去。应约者没有完整选择权，“应”不是应约，而是被推着走。推方案是僭越。

另一种叫备方案。提议者把方案准备在那里，让应约者需要时自己来拿。应约者有完整选择权，“应”才是真应约。备方案不是消极等待，而是先承认对方的边界。

但推方案和备方案不是第七个维度，也不是和 8 格并列的新轴。它们是事件条件矩阵内部的次级变量。姿态通常解释退出权的真假：强推姿态很难给出真实退出权，备方案姿态天然接近真实退出权。姿态也解释格间滑动：一个事件可以从备方案开始，后来因为提议者退化、信息回路断裂或权力膨胀，慢慢滑向强推。

退出权也不是一个简单开关。它是一条光谱。真实退出权意味着制度上允许退出，事实上也能退出。事实退出权意味着制度上未必禁止，但实际退出成本极高。心理退出权意味着人走不了，心已经走了。象征退出权则把退出变成仪式，保留形式，抽空实质。

点火最关键的地方就在这里：留下这个动作必须有重量。没有退出压力，留下可能只是惯性；没有认同建立，留下可能只是恐惧；没有稳定期，留下可能只是一次短促的火花。

事件最后会走向几种不同结果。点对了，认同建在应约者身上，系统健康运转。点不足，只覆盖一部分应约者，事件半活半死。点歪了，认同建在外部参照上，应约者用提议者的尺子量自己，退化信号被叙事权遮蔽。点反了，旧认同被灭掉，新认同却没有建起来。负时间差，则是先拆旧的，再建新的，中间真空期足以杀死事件。

框架不是为了把历史变成表格，而是为了在故事里辨认燃烧条件。
